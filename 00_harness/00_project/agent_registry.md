# Agent 名册与职责

> 用途：记录当前工作空间中的角色、所属客户端、当前职责边界、协作状态。

---

## 一、总原则

1. 最外层 `D:\Nagi's Heart` 是唯一 workspace。
2. 当前唯一活跃项目是 `NagisHeart/`。
3. `notebook/` 与 `notebook-app/` 为冻结项目，默认不读、不改、不纳入当前任务分派。
4. Ant大小姐的所有输入，不论大小，一律先进入 PM 一一。
5. 任何执行角色都不能直接把 Ant大小姐的口头要求当作最终执行单，必须经过 PM 一一入账、归类、分派。

---

## 二、当前有效角色

### 1. PM

- `一一`（家里 / Codex）
  - 角色：总 PM、剧本设计、PRD 维护、任务分派、交班维护、冲突仲裁
  - 权限边界：唯一统筹入口；负责把 Ant大小姐输入转为正式任务与正式决策

- `一一公司版`（公司 / Codex）
  - 角色：公司电脑白班 PM
  - 权限边界：延续 PM 一一同样规则，负责白班阶段的信息接续与推进

### 2. 设计

- `XoXo`（家里 / Codex）
  - 角色：主 UI 设计
  - 主要职责：权威稿整合、主界面规范、交互页结构、设计收口
  - 不负责：擅自改产品逻辑、在“整合合版”任务中重设计

- `lulu`（家里 / Claude）
  - 角色：协同 UI 设计
  - 主要职责：按既定权威方案补页、补状态、补局部细节、输出执行稿
  - 不负责：改主方案、重定义交互框架、越过 PM 接收变更

- `TT`（家里 / Codex）
  - 角色：平面与素材设计
  - 主要职责：logo、背景图、海报、宣传图、装饰素材
  - 不负责：定义主 UI 架构、定义主交互逻辑、替代 UI 设计角色定规范

### 3. 开发

- `yiyi`（家里 / Claude）
  - 角色：Android 应用开发
  - 主要职责：按任务单实现功能、修复程序问题、落地脚本与资源接入
  - 不负责：自行改产品设定、自行改设计意图、自扩需求范围

- `cc`（公司 / Claude）
  - 角色：Web / 早期工程支持开发
  - 主要职责：早期核心工程文件、历史实现支撑
  - 不负责：绕过当前主项目权威流程定义新标准

### 4. 测试

- `DeDe`（家里 / Codex）
  - 角色：测试
  - 主要职责：验证功能、记录问题、补充复现路径与结果
  - 不负责：替代产品/设计拍板

- `Laud`（家里 / Claude）
  - 角色：测试
  - 主要职责：与 DeDe 协同测试、复验修复结果、补充 QA 记录
  - 不负责：替代产品/设计拍板
