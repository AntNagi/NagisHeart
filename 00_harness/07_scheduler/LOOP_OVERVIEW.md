# 工作循环总览

本目录用于定义 `Nagi's Heart` 工作空间中的基础循环规则。

## 目的

工作循环是不同角色的固定工作回合。  
一个循环不是完整任务，也不是完整班次，而是一次受控的小闭环：

1. 读取当前状态
2. 执行一个受控小步骤
3. 回写结果
4. 停在可交班、可恢复的状态

## 当前基础循环

- `PM_LOOP.md`
- `WORKER_LOOP.md`
- `QA_LOOP.md`
- `SHIFT_HANDOFF_LOOP.md`

## 所有循环的共通规则

1. 默认活跃项目为当前 Git 仓库根目录。
2. `notebook/` 和 `notebook-app/` 为冻结项目，除非任务明确点名，否则默认不读、不改。
3. Ant大小姐的输入不由执行者直接处理，所有输入必须先经过 PM 一一并写入 harness 记录。
4. 每个循环只完成一个受控小步骤。
5. 每个循环结束时都必须留下书面结果，即使结果只是“阻塞”或“无变更”。
6. 如果某个循环改变了范围、优先级或权威文件，PM 必须在下一轮开始前更新对应的 harness 文件。

## 建议读取顺序

1. `..\00_project\PM_PROJECT_CONTEXT_BRIEF.md`
2. `..\00_project\WORKSPACE_STRUCTURE.md`
3. 相关的规划或治理文件
4. 角色对应的任务文件或交接文件

## 第一阶段建议触发节奏

- PM 循环：有新输入时、出现阻塞时、班次开始后、班次结束前
- 执行循环：每次只推进一个分配任务中的小步骤
- 测试循环：执行者声明完成一个步骤后，或进入验收前
- 交班循环：家里班和公司班切换前强制执行
