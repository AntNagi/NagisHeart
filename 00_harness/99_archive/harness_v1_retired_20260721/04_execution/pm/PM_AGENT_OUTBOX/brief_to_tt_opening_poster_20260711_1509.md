# 给 TT 的开屏主视觉设计 Brief

TT，请按下面的要求重新设计《Nagi's Heart》的开屏主视觉。这里说的是“开屏页”，不是进入游戏后的首页，也不是系统主页。

## 一、页面层级

开屏页是独立的第一张全屏海报：

`打开游戏 → 开屏海报 → START → 首页`

开屏页只承担“进入游戏”这一件事。开屏页不能放首页的“继续故事、章节目录、回忆画廊、系统设置”等主要操作。

## 二、画布与主体

- 画布必须是手机竖屏 9:16，建议输出 `1080 × 1920` PNG。
- 必须是整页海报构图，不是横图套进手机框，也不是普通 UI 页面。
- 以一张角色图作为主底图，保持人物完整、清楚、有视觉中心。
- 可以从这些现有素材中选择最适合的一张：`pillow`、`nagi_with_cat`、`birthday_crown`、`hug`、`remeet`。
- 不要做多图拼接、无边界融合或素材拼贴；这次要的是一张完整、有明确构图的原创开屏海报。
- 角色图只做必要的裁切、调色、暗部和文字可读性处理，不要改变角色脸部，不要添加黑眼圈或新的角色效果。

## 三、标题必须是海报的一部分

- `Nagi's Heart` 必须作为艺术字直接设计在海报里，并最终烘焙进 PNG。
- 不能依赖开发在图片上方用 HTML/Compose 浮一层普通字体。
- 标题要成为整张海报的主视觉设计元素：有明确字阶、字形气质、装饰关系和与人物/背景的构图关系。
- 标题可以使用英文品牌名 `Nagi's Heart`；这是品牌艺术字，不属于系统交互文本。
- 不要把标题做成随手放在画面上方的普通白色文字，也不要让它被人物脸部或背景线条完全吃掉。
- 若艺术字效果开发无法还原，必须交付已经包含标题的成品 PNG，而不是只交字体样式说明。

## 四、START 操作

- 开屏页只保留一个 `START` 操作。
- `START` 是操作文字，但要做成有设计感的字体按钮：通过字体、字距、大小、金色/银色层级、细线或小装饰来突出它。
- 不要使用长条背景框、胶囊按钮、切角按钮、卡片、深色按钮底或描边矩形。
- `Tap to start` 只是一行弱化的装饰提示，可以放在 START 下方；它不是第二个操作，也不能抢 START 的视觉权重。
- 不要出现 `Continue`、`Chapter`、`Gallery`、`Settings`、`Start` 菜单行或任何首页入口。

## 五、视觉方向

- 参考之前 demo 的“整页海报、标题与画面融合、底部动作提示”的布局逻辑即可，不要照搬它的粉色主题。
- 当前产品主视觉方向是深靛/炭灰、银白、低饱和暖金，可根据底图做克制的冷暖平衡。
- 不要强行做粉色乙女手游风，不要大面积粉色渐变、花瓣、爱心堆叠或甜腻装饰。
- 设计应有日乙游戏的精致感，但同时保留 Nagi 的冷感、留白和人物气质。
- 画面要有足够暗部承载底部 START，但人物和标题不能被压成一团黑。

## 六、交付物

请至少交付：

1. 一版主推开屏海报 PNG，尺寸 `1080 × 1920`。
2. 若需要比较，最多再给两版不同底图或构图；不要大量出相似小改版。
3. 每张图标明底图名称和版本号。
4. 同时说明哪一张是主推版。

## 七、验收标准

- 打开图片第一眼能判断这是《Nagi's Heart》的开屏海报。
- `Nagi's Heart` 是烘焙在图片里的艺术字，不是悬浮 UI 文本。
- 画面为完整 9:16 竖版，无横图适配感、无拼贴断边。
- 除 `START` 和弱化的 `Tap to start` 外，没有任何首页或系统操作。
- START 是文字型视觉操作，没有矩形/胶囊/卡片背景框。
- 标题、人物、START 三者的层级清楚，互不遮挡。
- 不是粉色主题；要与现有深色 UI 方向能够衔接。
- 最终 PNG 可以直接交给 yiyi 作为整页背景使用。

请先按这个 brief 出主推版，避免再把开屏页和首页混在一起。
