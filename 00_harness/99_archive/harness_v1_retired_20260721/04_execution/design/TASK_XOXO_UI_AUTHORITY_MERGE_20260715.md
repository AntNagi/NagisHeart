# XoXo 任务：UI 权威候选版严格合版

> 任务编号：`TASK-20260715-001`  
> 任务类型：整合合版  
> 优先级：P0  
> 负责人：XoXo  
> 核心规则：只做拼接，不做重设计

## 1. 任务目标

把已经由 Ant大小姐明确确认的页面，从三份现有设计稿中合成一份单一、可浏览、可追溯的 UI 权威候选版。

本任务不是探索设计，也不是补全设计。不得借合版之名修改视觉方向、重排信息层级、重新发明组件或补做尚未确认的页面。

## 2. 母版与来源

### A. 唯一母版

`design/NagisHeart_P0_HiFi_Design_XoXo_v2_0.html`

保留内容：

- 整体 UI 风格与设计语言
- light / dark 视觉体系
- HUD、人物对白、选项、LINE、存档、回忆画廊、设置、结局等母版中已成立的页面和组件
- 母版的布局、间距、字体层级、组件形态和交互动效基线

除下文明确允许替换或新增的页面外，母版现有内容不得改造。

### B. 缺页来源

`design/NagisHeart_Missing_Pages_Preview_XoXo_v1_0.html`

只拼入以下已通过页面：

1. 主页
2. 开场白页
3. 名字设置页
4. 大章开始页
5. 小节开始页
6. 弹窗体系，包括跳过本节确认弹窗

明确不采用以下页面：

1. 大章结束页
2. 小节结束页
3. 长旁白页
4. 剧情回顾页

大章/小节结束页的毛玻璃方案未通过；本任务不得修订后偷偷保留，也不得自行新做替代稿。

### C. 特定页面覆盖来源

`design/NagisHeart_LongNarration_StoryRecap_Redesign_Lulu_v1_0.html`

只覆盖：

1. 长旁白
2. 剧情回顾

保留 Lulu 版本的信息结构、阅读方式和页面形态；色彩调整为与母版一致的冷色深蓝灰体系，去掉偏暖、偏粉、偏米色的观感。色彩对齐是校准，不得借此改变布局和交互结构。

## 4. 合版执行要求

1. 三份源文件全部只读，不得覆盖。
2. 以 XoXo v2.0 复制出的新文件作为工作文件。
3. 每个拼入页面在 HTML 注释中记录来源文件与原页面名称。
4. 保持一个统一导航入口，能够逐页查看全部已纳入页面。
5. 页面名称、交互语义采用当前中文 PRD/交互口径；不要继承演示稿中过时的英文标签或旧 Menu 语义。
6. 对话正文不加中文引号；不得把旁白和人物对白混为同一组件。
7. 不改业务代码、剧本数据、背景映射和产品流程。
8. 发现来源冲突时暂停对应页面，只记录冲突并回 PM 一一，不自行判断。

## 6. 两轮自检

### 第一轮：来源自检

- 每个页面能否对应到本任务书指定来源
- 是否误用了 Missing Pages 的大章/小节结束页、长旁白或剧情回顾
- 是否误把 CoCo 交接中的错误判断带进权威版
- 是否覆盖了任一源文件

### 第二轮：越界自检

- 是否出现未经要求的新布局、新组件或新视觉方向
- 是否修改了已经确认正确的母版页面
- 是否自行完成了大章结束、小节结束或章节目录设计
- 长旁白和剧情回顾是否只做了冷色校准，没有重设计
- 输出是否完整可浏览，页面索引是否无遗漏

## 7. 完成与回报

完成后：

1. 将任务板状态从 `ready` 更新为 `review`。
2. 在合版记录中写清文件变更和待确认项。
3. 回报 PM 一一，不直接把候选版宣布为最终权威版。

只有 Ant大小姐完成视觉确认后，PM 才会把该文件提升为正式 UI 权威文件。
