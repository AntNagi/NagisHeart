## 3. 本轮必须收口的设计任务

### A. 首次进入 / 后续进入流程页

把以下流程对应页面全部收口成唯一规范：

首次进入：
`开屏页 -> 开场白页 -> 名字设置页 -> 第一章第一页剧情页`

后续进入：
`开屏页 -> 主页`

必须说明：
- 每一页的标题、说明、主操作文案
- 页面背景来源和层级
- 是否显示 HUD
- 点击热区和页面推进方式
- Continue Story 何时显示，何时不显示
- New Story 重置什么，不重置什么
- 首页“存档进度”的命名和入口样式
- 名字设置页本轮只保留玩家名输入，不保留 Nagi 称呼第二输入项

### B. 剧情 HUD 规范

必须按已确认口径输出唯一 HUD 规范：

- 左：返回
- 中：章节标题
- 右：自动播放 / 停止
- 右二：存档
- 最右：剧情回顾

需要定义：
- 默认态 / 点击态 / 禁用态
- 自动播放与停止的图标切换方式
- 点击存档后的“存档成功”反馈样式
- 剧情回顾面板的信息层级与关闭方式

不要再保留额外 Menu。

### C. 对白 / 短旁白 / 长旁白 三套展示规范

你要把三种文本展示明确区分开：

1. 人物对白
2. 短旁白
3. 长旁白

必须明确：
- 容器形态
- 字体风格差异
- 是否显示说话人
- 是否允许引号
- 单次点击推进规则
- 下一页引导样式

已确认规则：
- 不要代码自动加引号
- 长旁白不是一段一段点
- 长旁白整页展示，减少用户交互
- 长旁白超一屏时点击翻页，不滚动

### D. 连续叙事块规则

请把“什么时候算长旁白”写成可以执行的设计规范。

至少覆盖：
- 连续旁白块如何判定
- 1-2 段短旁白如何展示
- 3 段及以上如何进入长旁白
- 不足 3 段但排版超过底部 4 行时如何处理
- 长旁白分页时的视觉样式与翻页提示
- 不能把 `->` 这种内部跳转渲染成玩家可见选项

### E. 章节开始页 / 章节结束页

现在不是单层“每章”概念，而是双层章节结构都要有：
- 大章（篇 / 部）开始页、结束页
- 小章节（幕 / 节）开始页、结束页

两层都要有，且都是正式功能，不是装饰页。

你必须给出：
- 大章开始页必显信息
- 大章结束页必显信息
- 小章节开始页必显信息
- 小章节结束页必显信息
- 主操作 / 次操作
- 背景处理方式
- 是否显示 HUD
- “本章目录”“下一节”“下一章”“返回主页”等按钮什么时候出现

### F. 剧情回顾 / 章节目录 / 章节回看

这三个能力必须分开设计，不要再混成一个页面：

- 剧情回顾：当前剧情页内，查看“当前小章节从开头到当前句”的已读内容
- 章节目录：当前大章下全部小章节列表、顺序和状态
- 章节回看：对单个已经完成过的小章节进行完整回看

你必须明确：
- 三者入口分别在哪里
- 页面标题和返回路径
- 已完成 / 已跳过完成 / 进行中 / 未解锁 的状态样式
- 章节目录每项展示哪些信息
- 单个小章节回看时，和“剧情回顾”在视觉与信息层级上怎么区分

### G. 跳过本节

剧情页必须有“跳过本节”入口，但它不是顶部 HUD 五键之一。

你需要给出：
- “本节”正式定义为“当前小章节（section）”，不是整个大章
- 入口位置和视觉样式
- 点击后的确认样式
- 跳过后落到“小章节结束页”的状态说明
- 已跳过完成在章节目录中的显示方式

### H. 需要开发照着做的状态清单

请把下面这些关键状态都列成设计验收项：
- 首次进入无 Continue Story
- 有默认进度时才显示 Continue Story
- 手动存档成功提示
- 主页可见“存档进度”
- 剧情回顾展示“当前小章节开头到当前句”
- 已完成的小章节可单独回看
- 已跳过完成的小章节状态可识别
- 锁定章节 / 锁定结局的展示
- 自动播放开启 / 停止
- 长旁白分页中间页 / 末页

补充最新产品口径：
- `nagiCall` 本轮暂停，不设置，也不替换
- 手动存档槽位至少 10 个，且必须由玩家自己选择覆盖哪个槽位
- 因此本轮设计稿、状态图、验收项里都不要再保留 `nagiCall` 输入或相关交互
