# XoXo Full Scope Design Reply

## Summary

本回信按全量产品口径收口，不再按 MVP 或旧 HUD / 旧 Continue 逻辑理解。

本轮唯一目标是把以下内容锁成“yiyi 可以直接照着做，Laud 可以直接照着验”的规则：

- 首次进入 / 后续进入流程；
- 剧情 HUD 唯一结构；
- 人物对白 / 短旁白 / 长旁白三套展示规则；
- 连续叙事块与长旁白分页规则；
- 大章与小章节双层开始页 / 结束页；
- 剧情回顾 / 章节目录 / 章节回看三种能力拆分；
- “跳过本节”入口、确认与落点；
- “跳过本节”中的“本节”定义为当前小章节（section）；
- 已完成 / 已跳过完成 / 进行中 / 未解锁四种小章节状态；
- 手动存档卡槽数量、选槽保存与覆盖确认规则；
- 开屏、首页、系统页背景、标题系统、导航样式的视觉约束。

涉及具体视觉拍板的地方，统一标记为“待 Ant大小姐 确认”。

## Final Rules

### 1. 首次进入与后续进入

首次进入唯一流程：

`开屏页 -> 开场白页 -> 名字设置页 -> 第一小章节第一页剧情页`

后续进入唯一流程：

`开屏页 -> 主页`

唯一规则：

- 无默认进度时，主页不显示“继续故事”。
- 有默认进度时，主页显示“继续故事”，读取默认进度，不读取手动存档。
- “新的故事”只重置默认进度，从第一小章节第一页重新开始。
- “新的故事”默认不删除手动存档。
- 主页“存档”统一命名为“存档进度”。
- 名字设置页本轮只保留玩家名输入，不保留 `nagiCall` 或 Nagi 称呼第二输入项。
- 手动存档卡槽至少 10 个。
- 手动存档必须由玩家自己选择写入哪个槽位，不允许静默覆盖已有槽位。
- 目标槽位已有内容时，必须弹出覆盖确认。

### 2. 开屏页、首页与系统页

- 开屏页不是首页。
- 开屏页只有 `START`，没有其他主功能按钮。
- 首页才承载继续故事、新的故事、存档进度、章节目录、回忆画廊、系统设置等主入口。
- 首页标题必须与开屏页使用同一套标题系统，不允许开发重新手排一版简化标题。
- 首页、存档进度、章节目录、回忆画廊、系统设置、剧情回顾页，本轮统一共用开屏页同源背景图。
- 系统级页面全部使用中文交互文本；仅开屏艺术字与 `START` 可保留英文视觉识别。
- 所有主要操作采用文字型按钮，不使用厚重实体按钮框。
- 系统级页面开发实现不得再使用模糊描述，统一按固定色值与透明度执行。

### 3. 开屏页可读性处理

- 不允许在开屏主视觉上叠加黑色、灰色、白色半透明大遮罩块。
- 若标题或 `START` 可读性不足，只允许通过字体颜色、描边、柔阴影、轻发光、位置与字距微调解决。
- 不允许为了省事在画面中部铺一整块半透明底。

### 4. `nagiCall`

- 本轮暂停，不设置，也不替换。
- 任何设计稿、状态图、验收项中都不应再出现 `nagiCall` 输入项。
- 历史 script 中残留的 `{{nagiCall}}` 本轮不得以裸文本形式出现在玩家可见界面。

## Page Specs

### A. 开屏页

页面目标：

- 建立第一视觉；
- 给出唯一入口 `START`；
- 首次进入与后续进入都先经过此页。

必显信息：

- 锁定标题 `Nagi's Heart`
- `START`
- 装饰文案 `Tap to start`

背景来源：

- 使用已确认开屏主视觉基底图；
- 标题必须沿用锁定标题系统，不重排。

HUD：

- 隐藏

点击热区与推进：

- 轻触 `START` 或开屏安全区域进入下一步；
- 首次进入：进入开场白页；
- 后续进入：进入主页。

视觉硬规则：

- 无大面积遮罩；
- `START` 可做轻呼吸 / 轻发光；
- 标题不参与缩放动效。

待 Ant大小姐 确认：

- 标题细部样式是否维持当前版本；
- `START` 的最终描边 / 阴影强度。

### B. 主页

页面目标：

- 承接后续进入；
- 给出主功能入口；
- 建立“这不是开屏页”的层级分离。

必显信息：

- 同开屏页统一标题系统；
- 继续故事；
- 新的故事；
- 存档进度；
- 章节目录；
- 回忆画廊；
- 系统设置。

显示规则：

- 无默认进度时，不显示“继续故事”。
- 有默认进度时，显示“继续故事”。
- “新的故事”始终显示。

背景来源：

- 与系统级页面统一共用开屏同源背景。

HUD：

- 不显示剧情 HUD。

交互：

- 继续故事：进入默认进度所在剧情。
- 新的故事：二次确认后重置默认进度并进入开场白 / 名字设置 / 第一小章节。
- 其他入口进入对应系统页。

### C. 开场白页

页面目标：

- 建立产品语气；
- 首次进入时承担过渡。

必显信息：

- 当前句正文；
- `跳过`；
- 当前页进度提示可选。

背景来源：

- 与开屏 / 首页同源背景或其局部柔化版本。

HUD：

- 隐藏

交互：

- 一屏只显示一句正文；
- 点击正文安全区进入下一句；
- 点击 `跳过` 弹确认；
- 跳过后进入名字设置页。

文字规则：

- 不做大段文案堆叠；
- 不使用实体说明卡片；
- 可使用轻量局部模糊或小范围文字增强，但不加大块遮罩。

### D. 名字设置页

页面目标：

- 仅录入玩家名。

必显信息：

- 页标题；
- 玩家名输入框；
- 确认动作；
- 轻提示文案。

背景来源：

- 可与首页 / 系统页共用背景；
- 输入区居中，背景不抢焦点。

HUD：

- 隐藏

交互：

- 仅一个输入项；
- 合法输入后可进入故事；
- 确认入口为文字型按钮。

### E. 剧情页

页面目标：

- 承载正文阅读、选择与辅助操作。

必显信息：

- 当前剧情背景；
- 顶部 HUD；
- 底部对白 / 短旁白容器，或整页长旁白容器；
- 剧情层次级入口“跳过本节”。

背景来源：

- 来自剧情 BG mapping；
- 文字层和 HUD 必须服从阅读优先。

### F. 剧情回顾页

定义：

- 仅用于查看“当前小章节从开头到当前句”的已读内容。

页面目标：

- 在不离开当前阅读语境的前提下，快速回看本小章节已读内容。

必显信息：

- 页面标题：`剧情回顾`
- 当前小章节标题 / 序号
- 已读内容流
- 返回入口

背景来源：

- 与系统级页面统一背景；
- 内容阅读区为轻量阅读层。

HUD：

- 不显示剧情 HUD 五键；
- 以独立系统页形式打开。

交互：

- 从剧情页 HUD 最右“剧情回顾”进入；
- 关闭后返回当前剧情句位置；
- 不展示未来内容；
- 不补入“跳过本节”略过的未读内容。

### G. 章节目录页

定义：

- 展示当前大章下全部小章节结构、顺序与状态。

页面目标：

- 让玩家明确自己在当前大章中的位置；
- 提供进入进行中小章节或回看已完成小章节的入口。

必显信息：

- 页面标题：`章节目录`
- 当前大章标题 / 序号
- 小章节列表
- 每项状态

背景来源：

- 与系统级页面统一背景。

HUD：

- 不显示剧情 HUD 五键；
- 使用系统页导航。

每项至少展示：

- 小章节序号；
- 小章节标题；
- 状态；
- 进行中项的当前位置提示可选。

返回路径：

- 从剧情页内进入时，返回当前剧情页；
- 从主页进入时，返回主页；
- 从小章节结束页进入时，返回该结束页或进入下一步。

### H. 章节回看页

定义：

- 对单个已完成或已跳过完成的小章节进行完整回看。

页面目标：

- 提供章节级完整内容回顾；
- 与“剧情回顾”区分信息层级。

必显信息：

- 页面标题：`章节回看`
- 所属大章 / 小章节信息
- 完整已解锁内容
- 返回入口

规则：

- 只能从章节目录进入；
- 默认不覆盖默认进度；
- 默认不写入手动存档；
- 回看结束后返回章节目录。

### I. 存档进度页

页面目标：

- 承载手动存档列表与读取。

必显信息：

- 页面标题：`存档进度`
- 存档槽列表
- 空状态文案

背景来源：

- 与系统级页面统一背景。

规则：

- 手动存档槽位至少 10 个。
- 列表项至少包含：存档位、章节信息、时间、状态摘要。
- 点击“存档”后，必须先进入存档槽位选择，而不是直接自动写入某个固定槽。
- 选择空槽位时，直接保存并反馈成功。
- 选择已有内容槽位时，必须弹覆盖确认。
- 读取、覆盖、删除都需明确反馈。

### J. 回忆画廊页

页面目标：

- 承载回想与图鉴入口。

必显信息：

- 页面标题：`回忆画廊`
- 已解锁 / 未解锁状态
- 分类入口

背景来源：

- 与系统级页面统一背景。

### K. 系统设置页

页面目标：

- 承载文本、自动播放、音频、显示与数据管理。

必显信息：

- 页面标题：`系统设置`
- 文字速度
- 自动播放速度
- 背景音乐
- 显示模式
- 数据管理

背景来源：

- 与系统级页面统一背景。

### P. 系统级页面视觉数值

以下为直接给开发的固定值：

- 系统背景全局压暗层：`#132033`，透明度 `32%`
- 系统背景底部渐变：`#13203300 -> #132033CC`
- 系统浮层底色：`#1B2436`，透明度 `26%`
- 系统浮层描边：`#FFFFFF`，透明度 `10%`
- 系统页大标题：`#F5F1E8`
- 系统页一级文字：`#F4F1EA`
- 系统页二级文字：`#D6D2CB`
- 系统页弱信息：`#B7B3AD`
- 金色装饰点：`#D7BE86`

执行说明：

- 不再允许“背景别太白、浮层别太黑”这类模糊实现。
- 不允许继续使用过重的深灰大砖块浮层。
- 优先使用低对比、低厚度、轻托底关系。

### L. 大章开始页

必显信息：

- 大章序号；
- 大章正式标题；
- `轻触开始` 或 `轻触继续`。

背景处理：

- 使用该大章代表背景做柔化 / 降对比；
- HUD 隐藏；
- 标题优先。

主操作：

- 轻触开始 / 继续。

次操作：

- 无。

### M. 大章结束页

必显信息：

- 完成状态；
- 大章序号；
- 大章标题；
- 中文完成提示。

主操作：

- 有后续大章：`下一章`
- 无后续大章且未进入结局：`返回主页`
- 结局流按结局流程进入后续页

次操作：

- 条件满足时显示 `本章目录`

背景处理：

- 使用本章收束背景；
- HUD 隐藏。

### N. 小章节开始页

必显信息：

- 所属大章标识；
- 小章节序号；
- 小章节正式标题；
- `轻触开始` 或 `轻触继续`。

主操作：

- 轻触开始 / 继续。

背景处理：

- 使用本小章节首场景或指定图；
- HUD 隐藏。

### O. 小章节结束页

必显信息：

- 完成状态；
- 小章节序号；
- 小章节标题；
- 中文完成提示。

主操作：

- 有后续小章节：`下一节`
- 无后续小章节：进入大章结束页或按流程进入后续

次操作：

- `本章目录`
- 特定情况下 `返回主页`

特殊规则：

- “跳过本节”确认后必须落到这里；
- 若为已跳过完成，该页要明确展示“已跳过完成”状态文案或标识；
- 进入此页时更新默认进度，但不弹“存档成功”。

## HUD Specs

### 1. 唯一 HUD 结构

固定为：

- 左：返回
- 中：章节标题
- 右一：自动播放 / 停止
- 右二：存档
- 右三：剧情回顾

不保留额外 Menu。

### 2. 默认态 / 点击态 / 禁用态

#### 返回

- 默认态：轻量图标
- 点击态：短暂高亮
- 禁用态：仅在不可返回的强制演出中出现，平时应可用

#### 自动播放 / 停止

- 默认态：显示当前状态图标
- 点击态：切换并高亮反馈
- 禁用态：整页演出强锁时可禁用

切换规则：

- 开启自动播放后图标切为“停止”
- 点击“停止”后恢复“自动播放”

#### 存档

- 默认态：轻量图标
- 点击态：高亮并执行保存
- 禁用态：关键不可打断演出中可禁用

存档成功反馈：

- 使用轻量 toast 或短提示；
- 文案固定为：`存档成功`

#### 剧情回顾

- 默认态：轻量图标
- 点击态：进入剧情回顾页
- 禁用态：无

### 3. 导航视觉规则

- 顶部导航不采用厚重切角实体按钮排排站。
- 应使用弱容器、轻量图标、清晰标题、低存在感底板。
- 图标统一一套轻量线性风格，不混用不同粗细和不同容器形态。

### 4. 跳过本节

不是 HUD 五键之一。

入口要求：

- 位于剧情层次级操作区；
- 可放在底部对白区上缘、文本层角落或轻量浮动文字入口；
- 不能抢过 HUD 标题层级。

点击后：

- 弹确认；
- 这里的“本节”正式定义为“当前小章节（section）”，不是整个大章；
- 确认文案需明确“将直接结束当前小章节并进入本节结束页”。

跳过结果：

- 进入当前小章节结束页；
- 小章节状态记为“已跳过完成”；
- 在章节目录中标记为“已跳过完成”；
- 不把未播放正文写入剧情回顾。

## Text Presentation Specs

### 1. 人物对白

容器形态：

- 底部对白框

字体风格：

- 无衬线；
- 字重高于旁白；
- 偏暖白

说话人：

- 显示人物名

引号：

- 不自动添加；
- script 自带才显示

点击推进：

- 首次点击补完当前句；
- 再次点击进入下一句

下一页引导：

- 常规点击推进提示即可

### 2. 短旁白

容器形态：

- 底部旁白框

字体风格：

- 与对白明显不同；
- 可用衬线或斜体方向；
- 银灰色

说话人：

- 不显示人物名；
- 不显示“旁白”标签

引号：

- 不自动添加

点击推进：

- 与对白相同

### 3. 长旁白

容器形态：

- 整页毛玻璃阅读层

字体风格：

- 长文阅读字体；
- 左对齐；
- 行距宽于对白；
- 不建议整段斜体过度阅读疲劳

说话人：

- 不显示人物名；
- 不显示“旁白”标签

引号：

- 不自动添加

点击推进：

- 按页推进；
- 一页只需一次推进；
- 不逐段点；
- 不滚动

下一页引导：

- 非末页显示翻页图标与页码；
- 末页显示继续剧情引导

开发固定值：

- 正文字号：`16sp`
- 行高：`30sp`
- 字重：`Regular`
- 字间距：`0`
- 段间距：`18dp`
- 正文颜色：`#F4F1EA`，透明度 `92%`
- 正文阴影：`0 1 8`，颜色 `#0A0F19`，透明度 `18%`
- 阅读区宽度：屏宽 `78%`
- 阅读区位置：页面中部垂直居中
- 背景模糊：`18dp`
- 背景压暗层：`#101826`，透明度 `42%`

### 4. 连续叙事块判定

- 同一场景、同一时间、同一叙事节拍下连续无说话人文本，视为同一连续叙事块。
- 遇到人物对白、玩家选项、场景切换、时间切换、章节边界、脚本指定停顿时，必须结束当前块。

### 5. 长旁白判定

- 连续 1–2 段，且底部排版不超过 4 行：短旁白
- 连续 3 段及以上：长旁白
- 不足 3 段但底部排版超过 4 行：长旁白
- 长心理、重要情绪铺陈、说明性叙事可强制指定为长旁白

### 6. 长旁白分页

- 运行时按设备实际排版分页；
- 不滚动；
- 优先保留完整段落；
- 单段超页时，在句末标点处分割；
- 避免孤段、孤标题、孤尾；
- 自动播放也必须逐页推进。

## Chapter Start / End Specs

### 1. 双层章节结构

本产品不是单层章节。

必须同时存在：

- 大章（篇 / 部）
- 小章节（幕 / 节）

这两层都必须各自拥有：

- 开始页
- 结束页

### 2. 大章开始页

必显：

- 大章序号
- 大章正式标题
- 开始提示

HUD：

- 隐藏

### 3. 大章结束页

必显：

- 完成状态
- 大章序号
- 大章标题
- 中文完成提示

操作：

- 下一章 / 返回主页 / 结局流入口

HUD：

- 隐藏

### 4. 小章节开始页

必显：

- 所属大章
- 小章节序号
- 小章节标题
- 开始提示

HUD：

- 隐藏

### 5. 小章节结束页

必显：

- 完成状态
- 小章节序号
- 小章节标题
- 中文完成提示

主操作：

- 下一节

次操作：

- 本章目录
- 条件满足时返回主页

HUD：

- 隐藏

### 6. 状态与入口、返回路径

#### 未解锁

- 入口：无
- 返回路径：无
- 视觉：弱化 / `???`

#### 进行中

- 入口：章节目录项、继续故事
- 返回路径：返回当前剧情
- 视觉：高亮当前项

#### 已完成

- 入口：章节目录 -> 章节回看
- 返回路径：章节回看 -> 章节目录
- 视觉：完成标记

#### 已跳过完成

- 入口：章节目录 -> 章节回看
- 返回路径：章节回看 -> 章节目录
- 视觉：区别于普通完成，需明确“已跳过完成”标识

## Design Acceptance Checklist For Dev

- 首次进入只走：开屏页 -> 开场白页 -> 名字设置页 -> 第一小章节第一页剧情页。
- 后续进入只走：开屏页 -> 主页。
- 无默认进度时不显示“继续故事”。
- 有默认进度时才显示“继续故事”。
- “继续故事”读取默认进度，不读取手动存档。
- “新的故事”重置默认进度，不删除手动存档。
- 主页“存档进度”命名正确。
- 名字设置页只有玩家名输入。
- 开屏页只有 `START`，没有主页按钮。
- 开屏页无黑 / 灰 / 白半透明大遮罩块。
- 首页标题与开屏页标题系统一致。
- 首页、存档进度、章节目录、回忆画廊、系统设置、剧情回顾页统一共用开屏同源背景。
- 系统级页面交互文本全部中文。
- 顶部 HUD 固定为：返回 / 章节标题 / 自动播放或停止 / 存档 / 剧情回顾。
- HUD 中不再保留额外 Menu。
- 自动播放图标可切换为停止。
- 点击存档后出现“存档成功”反馈。
- 剧情回顾只展示当前小章节从开头到当前句的已读内容。
- 人物对白不自动加引号。
- 短旁白与人物对白视觉明显区分。
- 长旁白为整页阅读层，按页推进，不滚动。
- 长旁白非末页显示页码与翻页引导，末页显示继续引导。
- 不把 `->` 渲染为玩家可见选项。
- 不裸露 `{{playerName}}` 或历史 `{{nagiCall}}` 占位符。
- 大章开始页存在。
- 大章结束页存在。
- 小章节开始页存在。
- 小章节结束页存在。
- “跳过本节”存在，但不是 HUD 五键之一。
- 跳过本节确认后落到当前小章节结束页。
- 跳过本节中的“本节”被正确实现为当前小章节，不是整章。
- 小章节状态完整包含：未解锁 / 进行中 / 已完成 / 已跳过完成。
- 章节目录中能正确显示“已跳过完成”。
- 章节目录、剧情回顾、章节回看三种能力已拆开，不混页。
- 已完成小章节可回看。
- 已跳过完成小章节可识别并可回看。
- 剧情页顶部导航不是厚重切角实体按钮排排站。
- 手动存档槽位不少于 10 个。
- 手动存档由玩家主动选择目标槽位。
- 已有内容槽位被写入前会弹覆盖确认。

## Open Visual Decisions For Ant大小姐

- 开屏页锁定标题的最终阴影、描边和发光强度。
- `START` 的最终字体细节与呼吸动效强度。
- 系统级页面共用背景图是否继续锁定当前开屏同源图，或在同源范围内做局部裁切微调。
- 短旁白与长旁白的最终字体选择与具体字号。
- 长旁白阅读层的最终模糊半径、压暗比例与页码样式。
- 大章开始页 / 结束页、小章节开始页 / 结束页的最终装饰线、完成状态字样与中文操作文案细节。
- 剧情页顶部导航的最终图标样式与底板透明度。
