## Final Rules

### 1. 首次进入与后续进入

首次进入唯一流程：

```text
开屏页 -> 开场白页 -> 名字设置页 -> 第一章第一页剧情页
```

后续进入唯一流程：

```text
开屏页 -> 主页
```

规则：

- 开屏页不是首页。开屏页只承载首屏视觉和 `START`。
- 首次进入不显示 `Continue Story / 继续故事`。
- 只有存在默认进度时，主页才显示 `继续故事`。
- `继续故事` 读取上一次退出时保存的默认进度，不读取手动存档槽。
- `新的故事` 重置默认进度，从第一章第一页重新开始。
- `新的故事` 默认不删除手动存档。删除手动存档必须由玩家在数据管理或存档页主动触发，并有二次确认。
- 主页存档入口统一命名为 `存档进度`。

### 2. 默认进度与手动存档

必须分成两个产品概念：

| 类型 | 用途 | 触发 | 可见位置 |
|---|---|---|---|
| 默认进度 | 给 `继续故事` 使用 | 退出、节点推进、章节结束、进入下一章等自动维护 | 主页 `继续故事` |
| 手动存档 | 玩家主动保留某个进度点 | 剧情 HUD 点击 `存档` | 主页 `存档进度` |

HUD 存档必须保存：

- 当前节点；
- 当前句子 / 分页进度；
- 玩家名；
- 当前变量；
- 路线状态；
- 已读记录；
- 时间戳。

HUD 存档成功后显示 `存档成功`，并在主页 `存档进度` 中可见。

### 3. 语言与命名

- 系统级交互文本使用中文。
- 开屏页艺术字 `Nagi's Heart` 和 `START` 可作为视觉识别保留英文。
- 主页、HUD、章节、设置、存档、回顾、图鉴等正式交互文案全部中文。
- `{{playerName}}` 在正文、说话人、选项、回应、剧情回顾、存档摘要中都不得裸露。
- 一一对玩家固定称呼 `Ant大小姐`。
- `nagiCall` 本轮暂停：不设置，也不纳入本轮主链替换规则。
- `{{nagiCall}}` 的后续清理、脚本替换和产品称呼规则，移到后续 script / 产品清理项。

### 4. 内部跳转

- `->` 不能渲染成玩家可见选项。
- 单一自动跳转属于引擎内部行为。
- 玩家可见选项必须是有意义的选择、回应或确认，不显示后台 label。

## HUD Specs

剧情页 HUD 固定布局：

```text
左：返回
中：章节标题
右：自动播放 / 停止
右二：存档
最右：剧情回顾
```

不得保留旧 `Menu` 作为正式 HUD 入口。

### 1. 返回

- 默认态：低饱和浅色返回图标。
- 点击态：轻微提亮或按压透明度变化。
- 禁用态：仅在强制演出不可退出时出现，透明度降低，并不响应点击。
- 点击后：返回主页或打开确认，具体是否二次确认取决于当前是否有未写入默认进度的风险。

### 2. 章节标题

- 显示当前章节标题，不显示 debug node id。
- 长标题可截断或两行压缩，但不得遮挡右侧 HUD。
- 长按开启 debug 只允许开发 / QA 构建；玩家默认不可见。

### 3. 自动播放 / 停止

- 默认态：`自动播放` 图标。
- 开启后：图标切换为 `停止`。
- 点击 `自动播放`：进入自动播放状态。
- 点击 `停止`：停止自动播放，回到手动推进。
- 遇到选项、系统弹窗、剧情回顾、存档页时自动暂停。

### 4. 存档

- 默认态：存档图标。
- 点击态：轻微提亮 / 按压反馈。
- 点击后：写入手动存档。
- 成功反馈：屏幕中下部或 HUD 下方显示轻量 toast `存档成功`，1.2-1.8 秒后自动消失。
- 失败反馈：显示 `存档失败，请稍后重试`，并保留当前剧情状态。
- 存档成功反馈视觉：待 Ant大小姐 确认。

### 5. 剧情回顾

- 默认态：回顾图标。
- 点击后打开剧情回顾面板。
- 回顾内容：当前章节从开头到当前句的全部已播内容，包括人物对白、短旁白、长旁白已展示页、玩家选择及选择后的已展示内容。
- 不显示未来内容。
- 不显示后台变量、节点 ID、`->`、模板占位符。
- 关闭方式：右上角关闭、系统返回、点击面板外安全区域；关闭后回到原剧情位置，不推进剧情。
- 回顾面板视觉：待 Ant大小姐 确认。

## Text Presentation Specs

文本展示分三套：人物对白、短旁白、长旁白。

### 1. 人物对白

| 项 | 规范 |
|---|---|
| 容器 | 底部毛玻璃对白层 |
| 说话人 | 必显，显示解析后的角色名 |
| 字体 | 对白正体，无衬线方向 |
| 引号 | 代码不得自动加引号；script 写了才显示 |
| 行数 | 底部层最多 4 行，超过需编辑拆句或转长旁白 |
| 推进 | 逐字中首次点击补完；完整后再次点击推进 |
| 下一页引导 | 底部弱提示 / 小图标，具体视觉待 Ant大小姐 确认 |

### 2. 短旁白

| 项 | 规范 |
|---|---|
| 容器 | 底部毛玻璃旁白层 |
| 说话人 | 不显示人物名，也不显示 `旁白` 标签 |
| 字体 | 与对白明显区分，建议衬线 / 斜体方向 |
| 引号 | 不自动加；script 写了才显示 |
| 适用 | 1-2 段连续旁白，且实际排版不超过底部 4 行 |
| 推进 | 与对白一致：逐字补完后再次点击推进 |
| 下一页引导 | 与对白同位置，但样式可更轻 |

### 3. 长旁白

| 项 | 规范 |
|---|---|
| 容器 | 整页毛玻璃阅读层 |
| 说话人 | 不显示人物名，也不显示 `旁白` 标签 |
| 字体 | 长文阅读字体，建议衬线 Regular，避免整段斜体疲劳 |
| 排版 | 正文左对齐，行距更宽，保留段落间距 |
| 展示 | 一页作为一个完整阅读单位整体显示，不逐段点击 |
| 滚动 | 禁止滚动 |
| 分页 | 超一屏时按实际排版分页 |
| HUD | 剧情 HUD 保持可见可操作 |
| 推进 | 点击正文安全区翻页；末页点击继续剧情 |
| 自动播放 | 按当前页有效字数计算停留时间，逐页推进 |

### 4. 连续叙事块判定

连续无说话人旁白，且属于同一场景、同一时间、同一叙事节拍时，合并为连续叙事块。

遇到以下任一情况，结束当前叙事块：

- 人物对白或有说话人的独白；
- 玩家选项；
- 明确场景切换；
- 明确时间切换；
- 章节开始页；
- 章节结束页；
- script 指定停顿；
- 重要演出断点。

### 5. 长短旁白判定

- 连续 1-2 段，且底部旁白框实际排版不超过 4 行：短旁白。
- 连续 3 段及以上：长旁白。
- 不足 3 段，但底部旁白框实际排版超过 4 行：长旁白。
- 单段约 90 个中文全角字符：编辑预警为长旁白候选，但不是运行时硬切线。
- 长心理、重要情绪铺陈、说明性叙事可由 script 强制指定为长旁白，不受段数限制。

### 6. 长旁白分页规则

- 运行时以当前设备、字体缩放和安全区下的实际排版高度决定分页。
- 编辑预估目标为每页约 180-240 个中文全角字符，仅作预览参考。
- 优先保持完整段落。
- 单段超过一页时，在最接近页尾且不溢出的 `。！？……；` 等句末标点处分割。
- 避免标题、单行孤段或不足两行的段尾单独留到下一页。
- 非末页底部显示翻页图标和页码，例如 `1/3`。
- 末页显示继续剧情引导。
- 点击 HUD 按钮只执行 HUD 操作，不误翻页。

## Chapter Start / End Specs

每章必须有章节开始页和章节结束页，两者都是正式流程功能，不是装饰页。

### 1. 章节开始页

必显信息：

- 篇 / 章序号；
- 章节正式标题；
- `轻触开始` 或 `轻触继续`。

可选信息：

- 副标题；
- 时间 / 地点；
- 情绪引句。

可选信息没有内容时不保留空占位。

规则：

- 独立全屏页。
- 隐藏剧情 HUD。
- 使用本章首个场景或指定章节图作为背景。
- 背景需要柔焦、压暗或毛玻璃处理，保证标题优先。
- 点击非系统控件区域进入本章第一个剧情节点。
- 不显示选项、存档按钮、内部跳转箭头。

当前交付包 `start_flow/04_chapter_opening_9x16.png` 可作为视觉参考，最终视觉待 Ant大小姐 确认。

### 2. 章节结束页

必显信息：

- 完成状态，例如 `SECTION CLEAR`；
- 篇 / 章序号；
- 章节正式标题；
- 中文完成提示，例如 `本章完成`；
- 主操作。

主操作：

- 有后续章节：`下一章` 或 `下一幕`。
- 没有后续章节：`返回主页`。

次操作：

- 有章节目录功能且当前章节已可回看时，显示 `本章目录`。
- 无对应功能时不显示空按钮。

解锁信息：

- 有实际解锁内容时才显示解锁摘要。
- 不得为了填充页面展示虚假解锁信息。

规则：

- 独立全屏页。
- 隐藏剧情 HUD。
- 背景使用本章收束画面并做毛玻璃 / 降对比。
- 进入结束页时更新默认进度。
- 进入下一章后，默认进度定位到下一章开始页。
- 结束页自动记录默认进度，不属于手动存档，不显示 `存档成功`。
- 场景或小节结束不得冒充章节结束页。

当前交付包 `start_flow/05_section_clear_9x16.png` 可作为视觉参考，最终视觉待 Ant大小姐 确认。
