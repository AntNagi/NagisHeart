## 2. 直接开发的产品规则

### 进入流程

- 首次进入：`开屏页 -> 开场白页 -> 名字设置页 -> 第一小章节第一页剧情页`
- 后续进入：`开屏页 -> 主页`

### 主页

- 无默认进度时，不显示“继续故事”
- 有默认进度时，显示“继续故事”
- “新的故事”重置默认进度，不删除手动存档
- 主页入口文案固定为：
  - `继续故事`
  - `新的故事`
  - `存档进度`
  - `章节目录`
  - `回忆画廊`
  - `系统设置`

### 名字设置

- 本轮只保留玩家名输入
- 不保留 `nagiCall`

### HUD

唯一结构：

- 左：返回
- 中：章节标题
- 右：自动播放 / 停止
- 右二：存档
- 最右：剧情回顾

不保留额外 Menu。

### 跳过本节

- 必须有
- “本节”正式定义为当前小章节（section）
- 不是顶部 HUD 五键之一
- 点击后二次确认
- 确认后进入当前小章节结束页
- 章节目录中标记为“已跳过完成”
- 不把被跳过的未读正文写入剧情回顾

### 手动存档

- 手动存档槽位至少 10 个
- 必须由玩家自己选择写入哪个槽位
- 目标槽位已有内容时，必须弹覆盖确认
- HUD 存档成功后提示：`存档成功`

### 三个能力必须拆开

- `剧情回顾`：当前小章节从开头到当前句的已读内容
- `章节目录`：当前大章下全部小章节结构与状态
- `章节回看`：单个已完成 / 已跳过完成小章节的完整回看

不能混成同一个页面。

### 双层章节

同时存在：

- 大章（篇 / 部）
- 小章节（幕 / 节）

两层都必须各自有：

- 开始页
- 结束页

## 3. 直接开发的视觉 Token

### 系统级页面背景

- 背景基图：开屏同源图
- 全局压暗层：`#132033`，`32%`
- 底部渐变：`#13203300 -> #132033CC`
- 不额外叠白雾层

### 系统级页面浮层

- 浮层底色：`#1B2436`
- 浮层透明度：`26%`
- 浮层描边：`#FFFFFF`，`10%`

### 系统页文字

- 大标题：`#F5F1E8`
- 一级文字 / 一级按钮：`#F4F1EA`
- 二级信息：`#D6D2CB`
- 弱信息 / 空状态：`#B7B3AD`
- 金色装饰点：`#D7BE86`

### 长旁白页

- 正文字号：`16sp`
- 行高：`30sp`
- 字重：`Regular`
- 字间距：`0`
- 段间距：`18dp`
- 正文颜色：`#F4F1EA`，`92%`
- 正文阴影：`0 1 8`，`#0A0F19`，`18%`
- 阅读区宽度：屏宽 `78%`
- 阅读区位置：页面中部垂直居中
- 背景模糊：`18dp`
- 背景压暗层：`#101826`，`42%`
- 不加额外白色阅读卡片
- 不做厚重黑框

## 4. 不要做的事

- 不要再给开屏页叠黑 / 灰 / 白半透明大遮罩
- 不要把标题拆字手排成新版本
- 不要让系统页一部分有背景、一部分纯深底
- 不要把 `->` 渲染成玩家可见选项
- 不要裸露 `{{playerName}}` 或 `{{nagiCall}}`
- 不要做厚重切角实体按钮式顶部导航
