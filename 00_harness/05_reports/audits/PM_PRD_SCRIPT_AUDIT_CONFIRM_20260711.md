# PM 一一确认稿：PRD / Script / 当前实现差异巡检

> 状态：给 Ant 老板确认，确认前不分派给开发 / 测试 / 设计  
> 日期：2026-07-11  
> 范围：PRD、交互设计、script 数据结构、当前 Android 实现效果

## 1. 一一的当前判断

现在的问题不是单纯“界面还不够好看”，而是核心阅读体验、首进/续进逻辑、存档逻辑、HUD 口径、script 展示规则没有完全锁死，所以开发实现时按旧理解或局部理解走偏了。

当前实现只能跑出部分剧情流程，还没有达到全量正式产品的交付标准。尤其是：

- 台词被代码自动加了引号。
- 长旁白被拆成很多底部短句，用户需要一直点。
- `->` 这种内部跳转符号作为选项露给玩家。
- `{{nagiCall}}` 还会暴露。
- Continue Story 和存档逻辑混在一起。
- HUD 还是旧的 Auto + Save + Menu，不是最新要求的 Story Recap 直达。
- 每章开始页 / 结束页没有作为硬规则落进验收。

## 2. 建议锁定的产品规则

### 2.1 首次进入 / 后续进入

首次进入唯一流程：

```text
开屏页 -> 开场白页 -> 名字设置页 -> 第一章第一页剧情页
```

后续进入流程：

```text
开屏页 -> 主页
```

规则：

- 没玩过时，不显示“继续故事”。
- “继续故事”读取的是上一次退出时保存的默认进度。
- “继续故事”不是读取手动存档列表。
- “新的故事”重置默认进度，从第一章第一页重新开始。
- “新的故事”默认不删除手动存档，除非后面另做明确的删除确认。
- 主页“存档”改成“存档进度”。

### 2.2 存档

需要把两个概念分开：

- 默认进度：给“继续故事”用，自动记录上次退出 / 当前推进位置。
- 手动存档：玩家点 HUD 的存档按钮产生，能在主页“存档进度”里看到。

HUD 存档按钮必须做到：

- 手动存档槽位至少 10 个。
- 保存时进入手动存档槽列表，由玩家自己选择写入哪个卡槽。
- 若目标槽已有内容，必须弹出覆盖确认，不能自动覆盖。
- 保存当前节点、句子进度、变量、玩家名、路线状态。
- 点击后提示“存档成功”。
- 存档后能在主页“存档进度”列表查看。

### 2.3 剧情页导航栏

最新 HUD 口径建议锁为：

```text
左：返回
中：章节标题
右：自动播放/停止、存档、剧情回顾
```

规则：

- 自动播放按钮需要能切换为停止。
- 存档按钮保存当前手动存档，并提示成功。
- 剧情回顾展示当前小章节从开头到当前句的全部剧情。
- 不保留其他 Menu；HUD 最右固定为“剧情回顾”。
- “跳过本节”中的“本节”正式定义为“当前小章节（section）”，不是一句、不是一页、也不是整个大章。
- 点击“跳过本节”后必须二次确认，确认后直接进入当前小章节结束页。
- 跳过后当前小章节状态记为“已跳过完成”。
- 被跳过的未读正文不进入剧情回顾，但该小章节仍应在章节目录中可见并允许后续回看。

### 2.4 对话 / 旁白

规则：

- 对话文案不要由代码自动加引号。
- script 文本就是最终展示文本；真需要引号时，由 script 自己写。
- 短对话、短动作、短心理可以用底部对话框。
- 长旁白、长独白、情绪铺陈、连续旁白必须用整页 / 全屏展示。
- 同一个叙事段落内的连续旁白要合并成一个阅读块，减少点击次数。

### 2.5 选项 / 自动跳转

规则：

- `->` 不能作为玩家可见选项出现。
- 单一自动跳转应该是引擎内部动作，不应该渲染成选项卡。
- 玩家能看到的选项必须是有意义的选择或回应。
- `{{playerName}}` 必须在正文、说话人、选项、回应、剧情回顾、存档摘要里全部替换。

### 2.6 章节开始页 / 结束页

规则：

- 大章（篇 / 部）必须有开始页和结束页。
- 小章节（幕 / 节）也必须有开始页和结束页。
- “跳过本节”只允许跳过当前小章节，不提供一键跳过整个大章。
- 这是流程必需，不是视觉优化项。

## 3. 这次巡检发现的 PRD 缺口

原 PRD / 交互设计里已经提到过开场白、名字设置、Backlog、存读档、底部旁白、全屏旁白、章节过渡等概念。

真正缺的是“硬规则”：

- 没把“继续故事 = 默认进度”写死。
- 没把“第一次进入”和“后续进入”拆清楚。
- 没把“新的故事重置默认进度，但不删手动存档”写清楚。
- 没把 HUD 存档成功提示和主页存档列表写成验收项。
- HUD 仍有 Menu / Skip / Backlog 的旧口径混杂。
- 全屏旁白有组件概念，但没有作为 script 展示规则强制执行。
- 章节开始 / 结束页没有作为每章必备项。
- autoAdvance 的 `->` 没明确禁止出现在玩家界面。

## 4. 当前 script / 实现差异

已确认的数据和实现风险：

- story-data 里有多处 `autoAdvance: true`，其中一部分会配合 `->` label，容易露给玩家。
- story-data 里仍有大量 `{{nagiCall}}`，但 Ant大小姐 最新口径是本轮先不做称呼设置与替换链路，该项从当前开发主链移出，后续与 script 清理一并处理。
- `DialogueLayer.kt` 当前由代码给有 speaker 的台词加引号。
- `ChoiceLayer.kt` 会直接显示 choice label，所以 `->` 会被渲染。
- `NagiHud.kt` 当前右侧是 Auto + Save + Menu，不符合最新 HUD。
- Continue / Save 虽然已有代码函数，但 Laud 最新报告显示持久化和 Continue 行为仍需复验。

### 4.1 script 量化摸底

对当前 `story-data/nodes.json` 的只读扫描结果：

- 正文共 1668 条，其中无说话人旁白 762 条。
- 旁白按“被人物对白打断”做初步切分，共形成 306 个连续旁白块。
- 当前仅 11 条正文显式标记为 `fullscreen`。
- 按“连续 3 段及以上，或连续块累计约 90 字及以上”的编辑预判规则，有约 130 个连续块需要进入长旁白人工复核。
- 最长连续块达到 16 段 / 约 770 字，必须拆成多页，不能继续逐句底部点击。

以上 130 个是预判候选，不是最终自动改写结果。最终还要结合场景 / 时间切换、叙事节拍和设备实际排版确认。

### 4.2 当前数据结构缺口

当前 `DialogueLine` 只有 `id`、`speaker`、`text`、`display`，不足以稳定表达新的旁白规则。后续实现至少需要能表达：

- 显示类型：人物对白 / 短旁白 / 长旁白；
- 连续叙事块 ID；
- 编辑强制合并 / 强制断开；
- 长旁白段落边界；
- 可选的人工分页断点；
- 章节开始节点 / 章节结束节点及其展示字段。

运行时自动分页负责适配屏幕；人工分页断点只用于保护叙事节奏，不应要求 script 作者按具体机型手算字数。

## 5. Ant 已确认的产品口径

2026-07-11 已确认：

1. “新的故事”只重置默认进度，不删除手动存档。
2. `nagiCall` 本轮先不设置，也不替换；名字设置页只保留玩家名输入。
3. HUD 最右为“剧情回顾”，不再保留其他 Menu。
4. 连续叙事块合并，长旁白整页毛玻璃展示；超出一屏时点击分页，不滚动。
5. 每章开始页、结束页均为全量产品必备功能，并需要完整的信息展示规范。

## 6. 连续旁白、长短旁白与分页规则

### 6.1 连续叙事块边界

连续出现的无说话人旁白节点，且属于同一场景、同一时间、同一叙事节拍时，合并为一个连续叙事块。

遇到以下任一情况必须结束当前叙事块：

- 人物对白或有说话人的独白；
- 玩家选项；
- 明确的场景、地点、时间切换；
- 章节开始页、章节结束页；
- 必须单独停顿的系统演出或脚本指定断点。

编辑可通过显式标记强制合并或强制断开，避免算法破坏叙事节奏。

### 6.2 短旁白与长旁白判定

判定不能只依赖固定字数，采用“内容结构预判 + 设备实际排版复核”：

- 1–2 段且在底部旁白区实际排版不超过 4 行：短旁白。
- 连续 3 段及以上：长旁白。
- 不足 3 段，但实际排版超过底部旁白区 4 行：长旁白。
- 单段达到约 90 个中文全角字符时，编辑器预警为长旁白候选；最终仍以实际排版是否超过 4 行为准。
- 长心理、重要情绪铺陈、说明性叙事允许由 script 显式指定为长旁白，不受段数限制。

这样既保留 Demo 的“三段起进入长旁白”节奏，也避免不同屏幕、字号和标点下仅按字数误判。

### 6.3 三类文本展示区别

| 类型 | 容器 | 说话人 | 文字样式 | 推进方式 |
|---|---|---|---|---|
| 人物对白 | 底部毛玻璃对白框 | 必显 | 正体对白字体，不自动加引号 | 逐字中首次点击补完，完成后再次点击推进 |
| 短旁白 | 底部毛玻璃旁白框 | 不显示，也不显示“旁白”标签 | 使用独立旁白字体样式，与对白明显区分；参考 Demo 的斜体 / 叙事字形 | 与对白相同 |
| 长旁白 | 整页毛玻璃阅读层 | 不显示，也不显示“旁白”标签 | 长文阅读字体，左对齐、行距更宽；整页一次显示，不逐段点击 | 只按页推进 |

字体、字号、模糊半径、颜色由 Ant 最终确认；以上容器、信息层级和交互差异为产品硬规则。

### 6.4 长旁白分页

- 背景保留当前剧情画面并整页模糊 / 压暗，HUD 保持可见可操作，正文位于 HUD 下方安全阅读区。
- 不使用滚动；一页放不下时生成下一页。
- 分页以当前设备、当前字体缩放下的实际排版高度为最终依据，不用固定字数硬切。
- 编辑预估目标为每页约 180–240 个中文全角字符；该范围只用于脚本预览，不是运行时硬上限。
- 优先保持完整段落；完整段落放不下时移到下一页。
- 单段本身超过一页时，优先在 `。！？……；` 等句末标点处分割，选择最接近页面底部且不溢出的位置。
- 禁止把标题、单行孤段或不足两行的段尾孤立到下一页；必要时从前页回收一至两行。
- 每一页文本一次性呈现或整体淡入，不做逐段点击。
- 有后续页时，底部固定显示向下翻页图标和 `当前页/总页数`，例如 `1/3`；末页显示继续剧情的引导图标。
- 点击正文安全区进入下一页；点击 HUD 按钮只执行 HUD 操作，不误翻页。
- 自动播放时按本页有效字数计算停留时间，并受最短 / 最长阅读时间限制；自动播放也必须逐页展示，不跳过中间页。

## 7. 章节开始页 / 结束页信息规范

### 7.1 章节开始页

必显信息：

- 篇 / 章序号；
- 章节正式标题；
- `轻触开始` 或 `轻触继续` 操作提示。

可选信息：副标题、时间 / 地点、情绪引句。没有内容时不保留空占位。

交互与演出：

- 独立全屏页，HUD 隐藏。
- 使用本章首个场景或指定章节图作为背景，并做毛玻璃 / 降对比处理，保证标题优先。
- 点击任意非安全区控件位置后进入本章第一个剧情节点。
- 不出现选项、存档按钮或内部跳转箭头。

### 7.2 章节结束页

必显信息：

- 完成状态，例如 `SECTION CLEAR`；
- 篇 / 章序号与章节正式标题；
- 中文完成提示，例如 `本章完成`；
- 主操作：存在后续章节时显示 `下一章 / 下一幕`，没有后续章节时显示 `返回主页`。

条件信息与操作：

- 有章节目录功能时显示次操作 `本章目录`；没有该功能时不显示空按钮。
- 有实际解锁内容时才显示解锁摘要，禁止为了填充页面展示虚假信息。

交互与演出：

- 独立全屏页，HUD 隐藏，背景使用本章收束画面并做毛玻璃 / 降对比处理。
- 进入结束页时先写入默认进度；进入下一章后默认进度定位到下一章开始页。
- 结束页不是玩家手动存档，它不能替代 HUD 存档，也不应额外提示“存档成功”。
- 章节结束页只在整章结束时出现；场景或小节结束不得冒充章节结束页。

## 8. 给 Ant 确认的视觉基线建议

以下是结合 Demo 和当前 Android 字体 token 给出的建议基线，不作为 Ant 确认前的最终视觉拍板：

| 项目 | 建议基线 |
|---|---|
| 人物名 | 无衬线 Medium，13sp / 18sp；位于正文上方 |
| 人物对白 | 无衬线 Medium，18sp / 29sp；左对齐；最多 4 行；不自动加引号 |
| 短旁白 | 衬线 Italic，16sp / 28sp；左对齐；最多 4 行；无名称、无“旁白”标签 |
| 长旁白 | 衬线 Regular，18sp / 32sp；左对齐；段间距 20dp；水平安全边距 32dp；无名称、无“旁白”标签 |
| 长旁白毛玻璃 | 背景模糊建议 20dp，叠加约 48% 深色遮罩；实际按最亮 / 最暗 CG 做对比度复核 |
| 翻页提示 | 底部居中，下翻图标 + `1/3` 页码；距底部安全区 24dp；末页改为继续图标 |
| 章节开始页 | 篇章行 14sp，主标题 32sp / 42sp，操作提示 15sp；三者纵向居中排列 |
| 章节结束页 | 完成状态 14sp，章节标题 28sp / 38sp，完成提示 18sp；主次操作最多各 1 个 |

视觉验收补充：

- 长旁白正文区域必须在最小支持屏幕和系统最大支持字号下真实测量，不裁字、不滚动、不与 HUD / 翻页提示重叠。
- 毛玻璃不是单纯半透明色块，背景仍应能辨认当前剧情画面，但不得影响正文可读性。
- 长旁白不使用整段斜体，避免长时间阅读疲劳；与对白的区别由衬线字体、整页容器和更宽行距共同建立。
- 章节开始 / 结束页可以沿用 Demo 的柔焦构图，但具体色值、装饰线、字体和按钮外观由 Ant 最终确认。
