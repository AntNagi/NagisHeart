## 1. 开屏页校准

### 1.1 当前问题

- 标题仍像“贴片”，没有真正融进背景
- 仍出现白雾 / 半透明底块残留
- 标题与 `START` 的可读性处理方式不对

### 1.2 修正规则

- 不允许任何黑 / 灰 / 白半透明大底块压在主视觉上
- 标题 `Nagi's Heart` 必须直接压在背景上显示，背景保持完整
- 标题颜色：`#F6F3EE`
- 标题阴影：`0 2 12`，`#0B1220`，`28%`
- 标题描边：`#0F1726`，`14%`
- `START` 颜色：`#F4F1EA`
- `START` 装饰线颜色：`#D7BE86`
- `Tap to start` 颜色：`#D6D2CB`
- `START` 阴影：`0 1 10`，`#09111D`，`22%`

### 1.3 页面关系

- 开屏页主图、开场白页背景、名字设置页背景必须统一为同一张图
- 不允许开屏页一张图、开场白页另一张图、名字设置页又另一张图

## 4. 剧情页 HUD 校准

### 4.1 当前问题

- `投资的私心` 这节用了偏亮背景，但 HUD 和正文文字仍走深色，导致看不清
- 导航整体存在“图上再压一排深色控件”的感觉

### 4.2 HUD 固定值

- 顶部 HUD 底板：`#0F1827`，`18%`
- HUD 分割线：`#FFFFFF`，`10%`
- 标题文字浅色模式固定为：`#F4F1EA`
- 图标颜色固定为：`#D6D2CB`
- 图标点击态：`#F4F1EA`
- 顶部标题不跟随背景切成黑字

### 4.3 背景亮图处理

对亮背景章节，如 `投资的私心`：

- 额外叠一层顶部到中部的局部压暗渐变
- 渐变色：`#132033`
- 渐变透明度：顶部 `34%`，中部 `18%`，到底部 `0%`
- 目的：保证 HUD 与上部正文永远使用浅色字可读

## 5. 长旁白页校准

### 5.1 当前问题

- 字体气质不对
- 字号过大
- 只是“居中”，没有真正重做阅读版式
- 有些页面字色 / 背景层处理反了，出现浅底黑字或白底黑字式脏感

### 5.2 固定规则

- 长旁白正文一律使用浅色字，不允许黑字
- 正文字号：`16sp`
- 行高：`30sp`
- 段间距：`18dp`
- 字重：`Regular`
- 阅读区宽度：屏宽 `78%`
- 阅读区高度最大占比：屏高 `42%`
- 阅读区垂直居中
- 正文颜色：`#F4F1EA`，`92%`
- 正文阴影：`0 1 8`，`#0A0F19`，`18%`
- 模糊层：`18dp`
- 压暗层：`#101826`，`42%`

### 5.3 排版规则

- 阅读块整体在视觉中心，不上顶
- 不做“大白字铺满屏幕”
- 长旁白页中的 `轻触继续` 固定在底部安全区中心
- `跳过本节` 不与正文区重叠

## 6. 跳过本节入口与弹窗校准

### 6.1 当前问题

- `跳过本节` 与正文重叠
- 长旁白页面也出现同样重叠
- 弹窗样式过丑，像默认系统组件
- 弹窗文案错误：不能写“跳过后将直接进入下一节”，应进入当前小章节结束页

### 6.2 入口位置

#### 对白页 / 短旁白页

- 固定在右下安全区
- 距右 `20dp`
- 距底 `92dp`
- 不进入正文块宽度范围

#### 长旁白页

- 固定在右下安全区
- 距右 `20dp`
- 距底 `92dp`
- 永远低于阅读区底边 `32dp`

### 6.3 入口样式

- 文案：`跳过本节`
- 颜色：`#F4F1EA`
- 底板：`#F5F1E8`，`88%`
- 胶囊底板文字色：`#3B3B46`
- 圆角：`28dp`
- 内边距：横向 `18dp`，纵向 `10dp`
- 阴影：`0 8 24`，`#0A1120`，`18%`

说明：

- 这里允许“跳过本节”是独立胶囊轻按钮，因为它是次级危险操作，不属于主页文字按钮体系
- 必须弱化显示，避免误触

### 6.4 点击态

- 底板颜色变为：`#E8DEC9`
- 文字颜色变为：`#2C2B33`

### 6.5 确认弹窗样式

- 标题：`跳过本节`
- 正文：`确定跳过「当前小章节标题」？跳过后将直接进入本节结束页。`
- 弹窗底色：`#F6F2EC`
- 标题颜色：`#23242D`
- 正文颜色：`#4B4B58`
- 主按钮文字：`确定跳过`
- 次按钮文字：`取消`
- 主按钮颜色：`#8F6A42`
- 次按钮颜色：`#7A7A86`
- 弹窗圆角：`28dp`
- 宽度：屏宽 `84%`

禁止：

- 使用系统默认白色大弹窗
- 使用纯 Android 默认对话框按钮布局

## 7. 系统级页面统一校准

### 7.1 当前问题

- 主页 / 设置页背景暗度与浮层 token 没配套
- 开发改了背景暗度，UI 又单独给了浮层，导致对比关系失衡
- 有些页背景太白、浮层太黑
- 有些页浮层过重，像大块砖

### 7.2 统一规则

系统页只允许使用同一套关系：

- 背景基图：开屏同源图
- 全局压暗层：`#132033`，`32%`
- 底部渐变：`#13203300 -> #132033CC`
- 浮层底色：`#1B2436`，`26%`
- 浮层描边：`#FFFFFF`，`10%`

不允许：

- 单独再把背景提亮
- 单独再把浮层加深到 `40%+`
- 一页一套暗度

### 7.3 文字

- 大标题：`#F5F1E8`
- 一级文字：`#F4F1EA`
- 二级信息：`#D6D2CB`
- 弱信息：`#B7B3AD`
- 金色装饰：`#D7BE86`
