## 2. v1.1 调整说明

收到反馈后，本轮已从「组件检查器」改成「沉浸画面检查」。

主要调整：

- 去掉厚手机框，让 9:16 画面成为主体。
- 对话框不再做明显卡片，改成底部渐变叙事层。
- 选项不再做圆角卡片，改成细线文字带。
- LINE 模式去掉外层手机浮层，保留安静短句和气泡节奏。
- 章节地图去掉卡片容器，改成轻透明时间轴。
- 控制面板只作为检查工具，不代表游戏内 UI。
- 根据二次反馈，游戏画面内显性框线已全部去掉，改用渐变、留白、透明度和文字层级承载信息。
- 根据三次反馈，HUD 图标、选项文字、LINE 文本和章节文字的底框/按钮框也已弱化或移除，当前版本更接近纯图文叙事层。
- 根据四次反馈，当前版本增强了可读性和质感：检查面板默认收起，HUD 图标加入轻纹理，文字类元素加入淡金纹章 / 短线水印，避免“太弱”和“太简单”。
- 根据五次反馈，撤销背景图边缘模糊；标题、HUD、对话人名、选项文字改为“托字底纹”方案，使用五边形 / 锁感几何语言作为原创装饰，不直接复制官方 Logo。
- 根据六次反馈，LINE 和 Chapter 被定义为“被交互调出的特殊界面”，但二者不能共用同一套设计。LINE 改为朴素毛玻璃聊天层，Chapter 改为记忆档案式半透明时间轴层。
- 根据七次反馈，调出界面的毛玻璃层不能是硬边板块，已改为中心承托、边缘羽化消散的半透明毛玻璃。
- 根据八次反馈，Choice / LINE / Chapter 的毛玻璃中心强度提高，解决浅色背景和人物脸部背景上文字发虚的问题，同时保留软边。
- 根据九次反馈，HUD 标题与图标统一同一套颜色；LINE / Chapter 从高白度雾板调整为真正半透明毛玻璃，依靠 blur 和局部消息托底保证可读。
- 根据十次反馈，Choice 不能合成一个整体框，必须是三条独立选项。当前改为每条独立软毛玻璃托底 + 干净文字行 + 左侧小标记。
- 根据十一次反馈，统一形状语法：主要 UI 改为 Blue Lock 感的切角几何 / 五边形锁格语言，减少圆角和方块混用。
- 根据十二次反馈，移除所有语义不清的 X / 交叉点缀，小图标统一改为五边形或五边形环。

参考到的主流日乙倾向：

- CG / 立绘 / 背景优先，UI 不应把画面切成工具面板。
- 文本区多承载在底部叙事层、装饰条或轻透明区域中。
- 选项可以覆盖画面，但要服务紧张感和情绪分歧。
- HUD 通常轻量，不和剧情画面争抢视觉中心。

## 4. 本轮不检查什么

这不是最终高保真，也不是工程实现稿。

暂不检查：

- 最终 KV。
- 完整页面流。
- 真实 Android Compose 代码。
- 字体最终授权。
- 全部存档、图鉴、设置页细节。
- 背景最终 9:16 精裁切。

---

## 5. 重点检查问题

请只先看以下问题：

1. 图片是否终于成为主体，而不是被 UI 框住。
2. 文字是否和背景融在一起，但仍然清楚。
3. Light / Dark 是否只是保护阅读，而不是制造卡片感或框线感。
4. 玫瑰金点缀是否克制。
5. 选项是否有轻仪式感，但不再像按钮墙。
6. LINE 模式是否安静、低压、不像普通聊天软件仿真。
7. 章节地图是否有记忆档案感，且不提前剧透路线。
