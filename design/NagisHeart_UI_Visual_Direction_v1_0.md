# Nagi’s Heart · UI 视觉设计规范 v1.0

> 文档类型：UI 视觉方向 / Design System Brief  
> 使用对象：CoCo / CC / UI 设计窗口 / 后续实现方  
> 适用范围：移动端主版本、后续 Web 独立版本、主视觉 KV、游戏内 UI、章节地图、图鉴、存档、LINE 模式、结局页  
> 不包含：代码实现、Android / Web 技术架构、具体工程目录、最终高保真设计稿  
> 核心目标：建立一套能适配多背景、多剧情阶段、多情绪路线的 UI 视觉系统，而不是围绕单一主图或单一粉色主题做页面。

---

## 0. 一句话总纲

```text
Nagi’s Heart 的 UI 应该像冷白灯下的一封情书：
清冷、安静、透明，有足球世界的蓝灰压迫感，
也有私人恋爱里极克制的一点玫瑰金心跳。
```

本项目不是粉色少女恋爱 App，也不是黑蓝足球游戏界面。它是：

```text
冷白蓝灰基座上，偶尔浮出玫瑰金心跳的日乙视觉小说 UI。
```

---

## 1. 设计定位

### 1.1 产品类型

Nagi’s Heart 是一款 **移动端优先的日乙视觉小说**。

核心体验是：

```text
剧情阅读
→ 关键选择
→ 路线分歧
→ 结局达成
→ 图鉴 / 回想回收
```

UI 的首要任务不是展示功能数量，而是服务：

- 长文本阅读；
- 情绪沉浸；
- Nagi 的气质；
- 选择的重量；
- 多路线回收；
- 背景图切换后的稳定适配。

---

### 1.2 视觉核心

本项目视觉系统不应依赖单一主图。

正确逻辑：

```text
先定 UI 视觉系统
→ 用多张背景图压力测试
→ 再做产品主视觉 KV
→ 最后延展主页、游戏界面、章节地图、图鉴等页面
```

主图是品牌情绪样张，不是 UI 组件的唯一母版。

---

### 1.3 项目气质关键词

主关键词：

```text
冷光
蓝灰
清透
安静
克制
柔软
高级
低反应心动
私人空间
世界压迫感
```

禁止关键词：

```text
重粉色
国乙抽卡感
手游运营首页
满屏花瓣
少女贴纸
廉价玻璃拟态
黑金霸总风
赛博电竞风
网页模板感
按钮组件感
```

## 2. 为什么不能沿用旧粉色体系

旧版轻量网页可以使用粉玫瑰系，因为当时页面少、背景少、功能少。

但新版 Nagi’s Heart 有大量不同背景：

| 场景类型 | 视觉环境 |
|---|---|
| 蓝色监狱作战室 | 冷、暗、屏幕光、数据感 |
| 球场 / 世界赛 | 高对比、强光、运动感 |
| Ant 公寓 | 高级、柔和、生活感 |
| LINE 聊天 | 手机界面、近距离、低压 |
| 同居日常 | 暖、柔、亲密 |
| 曼城 / 媒体 | 冷、商业、国际化 |
| Bad 线 | 压抑、疏离、深色 |
| TRUE 线 | 冷白、淡金、完成感 |

如果 UI 绑定粉色体系，一旦进入作战室、球场、曼城、夜景、Bad 线，就会不适配。

因此，本次 UI 必须采用：

```text
中性基座 + 情绪点缀 + 明暗双模式 + 背景自适应
```

---

## 3. 视觉方向：冷光玫瑰

### 3.1 主方向名称

```text
冷光玫瑰 / Snowlit Rose
```

### 3.2 方向定义

```text
以冷白、蓝灰、深夜蓝作为 UI 基座，
以低饱和玫瑰金作为心动、亲密、选择与结局的情绪点缀。
```

它不是甜粉乙游，也不是纯黑蓝竞技风。

它应该同时成立：

- Nagi 的冷白、蓝灰、低反应；
- Ant 的克制、高位、柔软藏得深；
- 私人恋爱的温柔；
- 足球世界的压力；
- 媒体和资本的冷光；
- 结局分歧的情绪重量。

## 4. 色彩系统

### 4.1 基础色板

| 用途 | 色名 | HEX | 使用说明 |
|---|---|---:|---|
| 主背景冷白 | Snow White | `#F7F9FC` | 默认浅色背景、设置页、图鉴页基础底色 |
| 浅蓝灰底 | Mist Blue | `#E8EEF6` | 页面背景、章节地图底色、浅色面板 |
| 卡片白 | Soft Glass | `#FFFFFFCC` | Light Glass 面板底色，可带透明度 |
| Nagi 蓝灰 | Nagi Gray Blue | `#AEBAC8` | 描边、辅助图标、未选中状态 |
| 深夜蓝 | Deep Blue Night | `#101827` | 暗色模式、球场/作战室场景、深色遮罩 |
| 主文字 | Ink Navy | `#1B2430` | 浅色模式正文 |
| 次文字 | Cold Gray | `#6E7A89` | 说明、未激活、次级信息 |
| 心动点缀 | Muted Rose | `#C98A96` | 选择、心动提示、轻情绪装饰 |
| 高级点缀 | Rose Gold | `#BFA08A` | 标题线、选中描边、结局点缀 |
| Bad 线 | Dried Wine | `#7A3F4A` | Bad 线、危险状态、枯玫瑰氛围 |
| TRUE 线金光 | Pale Gold | `#D8C38A` | TRUE END、达成感、特殊彩蛋 |

---

### 4.2 色彩比例

默认 UI 建议比例：

```text
70% 冷白 / 蓝灰
20% 深夜蓝 / 冷灰
10% 玫瑰金 / 低饱和粉
```

玫瑰色不能铺满页面，只在以下位置出现：

- 游戏标题；
- 当前选中项；
- 心动类选项；
- 关键按钮高亮；
- 结局卡；
- 图鉴解锁点；
- 少量装饰线；
- 亲密剧情轻提示。

---

### 4.3 语义色，而不是固定粉色

不要规定“所有按钮都是粉色”。

应建立语义色：

| 语义 | 默认表现 | 变体 |
|---|---|---|
| 主文本 | Ink Navy | 深背景切换为冷白 |
| 次文本 | Cold Gray | 深背景切换为浅灰蓝 |
| 卡片底 | Soft Glass | 深背景切换为 Dark Glass |
| 当前进度 | Rose Gold 细线 | 深色场景降低亮度 |
| 心动反馈 | Muted Rose | 比赛场景弱化 |
| 危险状态 | Dried Wine | Bad 线可强化 |
| TRUE 状态 | Pale Gold | 与冷白同用 |
| 未解锁 | 蓝灰 / 雾化 | 禁止纯黑锁死感 |
| 已读 | 低饱和蓝灰 | 不要高亮过度 |

## 5. Light / Dark 双 UI 模式

### 5.1 为什么需要双模式

游戏背景可能非常亮，也可能非常暗。单一白色玻璃或粉色卡片无法适配所有背景。

必须设计：

```text
Light Glass：亮背景 / 生活背景
Dark Glass：暗背景 / 球场 / 作战室 / 夜景 / Bad 线
```

---

### 5.2 Light Glass

适用场景：

- 公寓；
- 卧室；
- 柠檬茶；
- 棒棒糖；
- 七夕；
- 夏日祭；
- 亲密日常；
- TRUE / GOOD 柔和场景。

视觉规则：

```text
半透明冷白底
轻蓝灰描边
深蓝黑正文
玫瑰金说话人名 / 当前选中态
轻模糊，但不能影响文字清晰度
```

建议：

```text
background: rgba(255,255,255,0.72-0.86)
border: 1px solid rgba(174,186,200,0.45)
text: #1B2430
accent: #BFA08A / #C98A96
```

---

### 5.3 Dark Glass

适用场景：

- 蓝色监狱作战室；
- 球场；
- 夜景；
- 曼城；
- 媒体 / 发布会；
- Bad 线；
- 深色 CG；
- 高对比背景。

视觉规则：

```text
半透明深蓝黑底
冷白 / 浅灰正文
银蓝描边
低亮玫瑰金点缀
背景加遮罩保证可读性
```

建议：

```text
background: rgba(16,24,39,0.68-0.82)
border: 1px solid rgba(174,186,200,0.28)
text: #F7F9FC
secondary: #C9D1DC
accent: #BFA08A
```

---

### 5.4 模式切换规则

每个剧情节点或背景应支持指定 UI 模式：

```text
uiTheme: light | dark | auto
```

若使用 auto，则按背景亮度 / 场景类型判断，但设计上必须允许手动指定。

默认建议：

| 场景 | 模式 |
|---|---|
| apartment | light |
| bedroom | light |
| lemontea | light |
| lolly | light |
| valentine | light |
| hug | light 或 dark，视背景亮度 |
| pitch | dark |
| drive | dark |
| afterwork | dark 或 mixed |
| openday | light 或 mixed |
| 作战室 / 数据屏 | dark |
| Bad 线 | dark |
| TRUE 线 | light + pale gold |

## 6. 背景适配规则

### 6.1 UI 不能抢背景

背景图是剧情氛围的一部分，UI 应该让玩家看见场景，而不是盖住场景。

但文字阅读优先级最高。若背景复杂，必须牺牲部分背景可见度来保证可读性。

优先级：

```text
文字可读性 > 点击可操作性 > 情绪氛围 > 背景完整展示 > 装饰
```

---

### 6.2 遮罩规则

游戏主界面应有可配置遮罩层。

建议：

| 背景类型 | 遮罩 |
|---|---|
| 明亮复杂背景 | 底部局部冷白渐变 + 轻模糊 |
| 暗色背景 | 底部深蓝黑渐变 |
| 人物 CG | 避开人物脸部，底部增强文本区 |
| 高对比球场 | 全屏轻暗化 + 对话框深色 |
| 亲密日常 | 柔光遮罩，降低锐度 |
| Bad 线 | 深蓝黑遮罩，局部枯玫瑰点缀 |
| TRUE 线 | 冷白柔光，淡金点缀 |

---

### 6.3 背景压力测试清单

UI 设计必须至少用以下背景进行测试：

| 文件 | 场景类型 | 测试目的 |
|---|---|---|
| `apartment.jpg` | 高级公寓 | 浅色生活场景适配 |
| `bedroom.jpg` | 亲密日常 | 暖柔背景下文字可读 |
| `pitch.jpg` | 球场 / 竞技 | 深色 / 高对比场景适配 |
| `openday.jpg` | 蓝锁开放日 | 角色亲近 + 环境适配 |
| `valentine.jpg` | 高甜 / 玫瑰感 | 避免过粉、过腻 |
| `hug.jpg` | 亲密情绪 | UI 不抢人物情绪 |
| `drive.jpg` | 夜景 / 深色 | Dark Glass 可读性 |
| `afterwork.jpg` | 都市 / 成年感 | 克制高级感 |
| `curry.jpg` | 日常轻喜剧 | 生活场景稳定性 |
| `propose.jpg` | 终局 / 特殊彩蛋 | TRUE / 特殊演出适配 |

验收标准：

```text
换背景后，文字仍然清楚。
图标不突兀。
按钮不廉价。
Nagi 气质不被 UI 抢走。
恋爱场景不冷，比赛场景不粉。
```

## 7. 字体与排版

### 7.1 字体气质

字体应体现：

```text
日乙感
高级感
清透感
长文本阅读舒适
```

建议方向：

- 标题：细衬线 / 高级感 serif / 轻奢感；
- 英文标题：细 serif 或优雅现代字体；
- 正文：高可读无衬线；
- 系统小字：清晰、克制、不要装饰化。

禁止：

- 过度可爱手写体；
- 厚重黑体标题；
- 赛博科技字体全局使用；
- 大量花体英文；
- 低可读日系装饰字体。

---

### 7.2 标题层级

建议字号层级以移动端为准：

| 层级 | 用途 | 建议 |
|---|---|---|
| H1 | 游戏名 / 结局名 | 高级 serif，字距略放 |
| H2 | 篇章标题 | 冷白 / 深蓝 + 细线装饰 |
| H3 | 节点标题 | 简洁，少装饰 |
| Body | 正文对话 | 阅读优先 |
| Caption | 时间、状态、说明 | 小而清楚 |

---

### 7.3 正文阅读规则

正文是核心，必须稳定。

要求：

```text
正文行高充足。
每行不要过长。
深浅背景都要有足够对比。
对话框内留白不能太挤。
不要为了美观牺牲可读性。
```

建议：

- 正文行高：1.45–1.65；
- 对话框内边距：上下不少于 16，左右不少于 18；
- 角色名和正文有明确层级；
- 点击提示极弱，不干扰阅读；
- 逐字播放时文字不跳动、不重排。

## 8. 组件视觉规范

### 8.1 HUD

HUD 应极简，不做手游运营栏。

允许常驻：

- 返回 / 菜单；
- 当前章节名；
- 存档入口；
- Auto；
- Skip；
- Backlog。

不允许常驻：

- 好感条；
- 理解度；
- 距离值；
- 控制值；
- 路线倾向；
- 结局资格；
- 大量任务 / 活动入口。

HUD 视觉：

```text
细线图标
低透明度
默认蓝灰 / 银灰
激活时玫瑰金
不抢主画面
```

---

### 8.2 图标

图标必须中性。

建议：

```text
细线
单色
圆角轻微
默认蓝灰 / 银灰
选中态玫瑰金
```

禁止：

```text
粉色圆底 icon
可爱贴纸图标
粗填充图标
国乙手游图标感
过度拟物
花瓣 / 爱心滥用
```

功能图标建议：

| 功能 | 图标气质 |
|---|---|
| Backlog | 细线文档 |
| Save | 细线书签 / 存档卡 |
| Gallery | 细线相框 |
| Settings | 细线齿轮 |
| Auto | 极简播放符 |
| Skip | 双箭头细线 |
| Chapter | 细线时间轴 |
| LINE | 简洁气泡，不要社交软件仿真过强 |

---

### 8.3 对话框

对话框是最重要组件。

必须提供：

```text
Light Glass
Dark Glass
```

基本结构：

```text
说话人名
正文
点击提示
可选：当前模式极弱装饰线
```

规则：

- 说话人名不能太大；
- 正文优先可读；
- 对话框高度不要固定死，应适配文本；
- 不要过度圆润到像儿童 App；
- 不要过度玻璃拟态导致文字发虚；
- 不要使用大面积粉色底。

---

### 8.4 旁白

旁白分两种。

#### 底部旁白

用于短动作、心理、过渡。

视觉：

```text
与普通对话框同体系
可不显示说话人名
文字略柔
```

#### 全屏旁白

用于连续叙事、情绪铺陈、场景转换。

视觉：

```text
全屏轻遮罩
中部文本卡 / 中部文本区域
文字居中或左对齐均可，但必须阅读舒适
背景可见但弱化
```

全屏旁白不能像系统弹窗，也不能像网页文章。

---

### 8.5 选项卡

选项是情绪分歧点，应有仪式感。

视觉：

```text
半透明长卡片
细线描边
轻微上浮
选中态玫瑰金细线亮起
按下时轻微收缩或暗化
```

禁止：

```text
大面积粉色按钮
战斗技能按钮感
纯系统列表感
粗边框
高饱和渐变
```

选项标签可出现，但要克制：

```text
心动
理解
认真
靠近
沉默
推他一把
让他自己决定
```

标签不要喧宾夺主。

---

### 8.6 LINE 模式

LINE 模式是特殊演出，不是普通 VN 对话框换皮。

视觉规则：

```text
手机界面浮层
玩家与 Nagi 左右气泡
时间间隔感
输入中状态
已读 / 未读节奏
轻微手机屏幕感
```

颜色：

| 对象 | 气泡 |
|---|---|
| Nagi | 冷灰蓝 / 深灰蓝 |
| 玩家 | 低饱和玫瑰白 / 冷白 |
| 系统时间 | 低透明蓝灰 |
| 输入中 | 三点动画，克制 |

禁止：

- 可爱聊天贴纸风；
- 明亮绿色聊天软件仿真；
- 表情包堆满；
- 过度社交 App 化。

LINE 的核心气质：

```text
低压、安静、短句、精神补位。
```

---

### 8.7 章节地图

章节地图应像：

```text
记忆档案 / 心跳轨迹 / 路线档案
```

不应像：

```text
RPG 大地图
手游活动地图
抽卡游戏章节页
```

建议结构：

- 纵向时间轴；
- 篇章卡片；
- 节点点位；
- 已读亮起；
- 未解锁为 `???`；
- 当前节点有细线光晕；
- 关键分歧点有玫瑰金标记；
- Bad / Dream / Stay 路线不提前完全暴露。

---

### 8.8 存档页

存档页应清楚、克制、可快速操作。

每个存档卡包含：

- 章节名；
- 节点名；
- 保存时间；
- 背景缩略图或色块；
- 当前阅读进度；
- 读取 / 覆盖 / 删除操作。

视觉：

```text
卡片式
冷白 / 深蓝双模式
缩略图加遮罩
不要手游仓库感
```

---

### 8.9 图鉴页

图鉴应像：

```text
相册
记忆档案
结局记录
```

结构：

- CG 图鉴；
- 结局图鉴；
- 事件回想；
- LINE 回想；
- 特殊彩蛋。

未解锁状态：

```text
雾面遮罩
???
细线锁定图标
不要黑块粗暴遮挡
```

已解锁状态：

```text
轻微亮起
可点击回看
显示篇章 / 标题 / 解锁时间
```

---

### 8.10 结局卡

结局卡必须有仪式感。

结局视觉基调：

| 结局 | 色调 |
|---|---|
| TRUE END 世界第一，与你 | 冷白 + 淡金 |
| GOOD END 那么完美，那么爱你 | 白玫瑰 + 浅粉金 |
| NORMAL END 普通情侣 | 暖灰 + 柔白 |
| BAD END 好麻烦 | 深蓝黑 + 枯玫瑰红 |

结局卡不能像普通弹窗。  
它应像一张完成回收的记忆卡，具有终局感。

---

## 9. 页面级视觉方向

### 9.1 主页

主页是品牌第一印象。

要求：

```text
Nagi 是绝对视觉中心。
标题 Nagi’s Heart 清晰。
按钮少而精。
不要运营入口。
不要任务 / 活动 / 商城。
```

主页可包含：

- Start / Continue；
- Chapter；
- Gallery；
- Settings；
- 开场白回看。

视觉：

```text
冷白蓝灰为主
玫瑰金标题或细线
背景可使用主视觉 KV 或动态氛围图
UI 留白足够
```

---

### 9.2 游戏主界面

核心目标：

```text
长时间阅读不疲劳。
背景切换不崩。
文字永远清楚。
```

不追求强 UI 存在感。

---

### 9.3 设置页

设置页应简洁，不要系统后台感。

分类：

- 文本；
- 音频；
- 显示；
- 跳读；
- 数据。

视觉保持冷白 / 蓝灰基座。

---

### 9.4 Web 版视觉

Web 后续独立实现，但视觉上不应重做成横向网页游戏。

建议：

```text
桌面背景 + 居中手机画布
```

手机画布内保持移动端体验。  
桌面背景可从当前剧情背景中模糊延展，或使用冷光玫瑰主题背景。

---

## 10. 主视觉 KV 的边界

### 10.1 主图仍然需要

主图用于：

- 首页第一印象；
- 品牌宣传；
- 标题气质；
- 视觉系统样张；
- 小红书 / 封面延展。

---

### 10.2 主图不能绑架 UI

禁止：

```text
主图是粉色 → 全 UI 都做粉色
主图是深蓝 → 全 UI 都做黑蓝
主图有花 → 全 UI 都铺花瓣
```

主图应服从 UI 视觉系统，而不是反过来定义全部组件。

---

### 10.3 主图建议方向

```text
冷光玫瑰
竖屏 9:16
Nagi 是绝对中心
冷白蓝灰基调
少量玫瑰金心动点缀
可留出标题与首页按钮空间
不重绘人物优先
不做强情侣图
不做热血比赛图
不做粉色拼贴图
```

主图情绪建议：

```text
他已经在你的世界里了，
但还没有完全把心交出来。
```

---

## 11. 动效原则

动效应低声、克制、服务阅读。

建议：

| 场景 | 动效 |
|---|---|
| 背景切换 | 轻淡入，不强闪 |
| 对话框出现 | 短淡入 |
| 选项出现 | 轻上浮 + 淡入 |
| 选项选中 | 玫瑰金细线亮起 |
| 章节标题 | 慢一点，有仪式感 |
| 图鉴解锁 | 轻提示，不做手游弹窗 |
| 结局达成 | 更强一点，但不浮夸 |
| LINE 输入中 | 三点微动 |

禁止：

- 大量粒子常驻；
- 花瓣全局飘；
- 按钮弹跳过强；
- 强光效闪烁；
- 手游抽卡式爆光；
- 转场过慢影响阅读。

---

## 12. 装饰元素

允许元素：

```text
细线
微光
玻璃边缘
雾面遮罩
淡金线
低饱和玫瑰色点
数据线 / 屏幕光
轻微雪光 / 冷光
```

谨慎使用：

```text
花瓣
爱心
星星
玫瑰
蝴蝶结
高饱和渐变
```

禁止全局常驻：

```text
花瓣循环动画
满屏粉色纹理
大面积爱心
少女贴纸边框
```

花瓣 / 玫瑰只用于特定剧情：

- 七夕；
- 生日；
- 情人节；
- 告白；
- TRUE END；
- 求婚彩蛋。

- 奢侈品广告感；
- 霸总办公室风；
- 过重金属质感；

Ant 是高位、克制、柔软藏得深，不是黑金霸总。

## 15. 验收标准

### 15.1 基础验收

```text
文字清楚。
按钮可点。
页面不挤。
背景不抢文字。
UI 不抢 Nagi。
浅色背景和深色背景都能适配。
```

---

### 15.2 风格验收

```text
整体不像国乙手游。
不像抽卡页面。
不像粉色少女 App。
不像黑蓝足球游戏。
不像网页模板。
有 Nagi 的冷白蓝灰。
有 Ant 的克制高级感。
有私人恋爱的玫瑰金心跳。
```

---

### 15.3 场景验收

```text
公寓场景：柔软但不甜腻。
球场场景：有压力但不电竞。
作战室场景：冷光数据感成立。
LINE 场景：亲近、低压、短句感成立。
Bad 线：压抑、疏离，但不廉价黑红。
TRUE 线：完成、清透、淡金，不浮夸。
```

---

### 15.4 产品一致性验收

```text
换背景后，UI 仍像同一个作品。
换章节后，组件不突兀。
换路线后，情绪能变化，但设计系统不散。
主图、主页、游戏界面、图鉴页属于同一套视觉语言。
```

## 17. 当前结论

Nagi’s Heart 的 UI 设计路线应确定为：

```text
不依赖单一主图。
不继承旧粉色体系。
以冷白蓝灰为中性基座。
用玫瑰金做克制心动点缀。
提供 Light / Dark 双模式。
用多背景压力测试保证剧情适配。
```

最终目标：

```text
让每一张背景都能承载剧情，
让每一次选择都有心跳，
让玩家始终觉得：这是 Nagi’s Heart，
而不是某个通用乙游模板。
```
