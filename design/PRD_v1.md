# Blue Lock: Nagi's Heart — 产品需求文档 (PRD v1)

> 基于现有网页版的完整功能还原，作为 Android 重构的基准文档。

## 1. 产品概述

**产品名称**: Blue Lock: Nagi's Heart · 梦向乙游  
**类型**: 文字冒险 / 乙女向恋爱养成  
**IP 来源**: 《蓝色监狱》(Blue Lock) 二创  
**核心角色**: 凪诚士郎 (Nagi Seishiro)  
**玩家角色**: Ant — 年轻女性投资人，玩家可自定义名字  
**辅助角色**: 一一 (YiYi) — Ant 公寓的 AI 智能管家  
**目标用户**: 蓝色监狱女性粉丝，乙女游戏爱好者

## 2. 核心玩法

### 2.1 对话驱动的叙事
- 游戏以**对话气泡 + 旁白**的形式推进剧情
- 角色类型：`nagi`（凪）、`ant`（玩家）、`yiyi`（一一）、`sys`（旁白/系统叙述）
- 文字逐字打印（typewriter effect），打印速度 26ms/字
- 玩家点击/轻触屏幕推进下一句对话

### 2.2 旁白阅读模式
- 当连续出现 **≥3 句**旁白（sys）时，自动进入**沉浸式旁白面板**
- 旁白以居中蒙版浮层形式呈现，多段落排列
- 轻触关闭，回到对话推进

### 2.3 选项分支系统
- 每个场景节点的对话结束后，弹出**选项卡片**供玩家选择
- 选项通常 1-3 个，分为不同类型标签：
  - `心动` — 偏感性/浪漫的回应
  - `理性` — 偏冷静/分析的回应
  - `作` — 傲娇/撒娇/试探的回应
- 选择后触发：
  - Ant 的台词播放
  - Nagi 的回应台词序列
  - 数值变化
  - 可能的路线分岐

### 2.4 称呼替换系统
- 玩家在首次游玩时录入两个名字：
  - **玩家名字** — 替换所有文本中的 `Ant`
  - **给 Nagi 的昵称** — 替换所有文本中的 `凪诚士郎` / `凪` / `Nagi`
- 替换逻辑：长名优先匹配，替换后不二次扫描（防止「凪宝」→「凪宝宝」）

## 3. 数值系统

### 3.1 显性数值
| 数值 | 说明 | UI 表现 |
|------|------|---------|
| bond (羁绊) | 选项中 b 值累加 | HUD 右上角心形进度条 |

### 3.2 隐性核心数值
| 变量 | 含义 | 影响 |
|------|------|------|
| `I` | 理解度（懂他程度） | 累加自选项 `u` + `i` |
| `D` | 心墙（距离/疏离感） | 累加自选项 `d`，D≥6 触发 Bad End |
| `ego` | Nagi 自主意志 | 影响路线解锁 |
| `control` | Ant 管控倾向 | 影响路线解锁 |

### 3.3 路线标记 & 旗标
| 变量 | 含义 |
|------|------|
| `mj` | M（默契线）或 J（较劲线），决定第七部的分支内容 |
| `path` | `dream`（梦想线）/ `stay`（相守线）/ `cling`（执念→Bad End） |
| `route` | 同 path，用于章节目录中的路线筛选 |
| `finalFlag` | 终局关键选择（support / waver 等） |
| `nagiResonate` | Nagi 是否觉醒共鸣（True End 必要条件） |
| `antCompress` | Ant 是否压迫/控制过 Nagi |
| `witnessFlag` | 是否见证关键时刻 |
| `personalHonor` | 是否尊重 Nagi 个人荣誉 |
| `nagiNameIndependent` | Nagi 是否建立独立自我认同 |

### 3.4 其他隐性计数器（约 18 个）
`habitWarm`, `habitDepend`, `ordinaryHappiness`, `publicCoupleExposure`, `antSoft`, `nagiCare`, `egoHold`, `loveNotHabit`, `distance`, `badRisk`, `possess`, `controlSeed`, `nagiSelf`, `antIdealize` 等 — 微调路线走向和结局判定。

## 4. 剧情结构

### 4.1 八部章节
| # | 章名 | 幕数 | 核心内容 |
|---|------|------|----------|
| 第一部 | 原作前置篇 | 6 幕 | 初遇、选拔、心意相通、U-20 |
| 第二部 | 关系确立篇 | 5 幕 | 开放日、确立关系、公寓邀请 |
| 第三部 | NEL · 低谷 | 7 幕 | 巅峰到落选、咖喱饭、生日、拥抱、亲密、Side-B 复归 |
| 第四部 | 世界杯 | 4 幕 | 追加名单、训练、生死局、豪门来信 |
| 第五部 | 世界杯后·夏天 | 12 幕 | 同居日常、七夕、夏日祭、签约 |
| 第六部 | 豪门新星篇 | 7 幕 | 曼城生活、训练、采访、读书之秋、万圣夜、飙车 |
| 第七部 | 冬季分线篇 | 6 幕 | 经纪公司发布、M线/J线分支、醉酒 |
| 第八部 | 世界中心篇 | 17 幕 | 三条路线终局（Dream/Stay/Bad）+ 四个结局 |

**总计**: ~64 个可游玩幕 / ~200+ 场景节点

### 4.2 路线体系
```
                    ┌── Dream 线（梦想线）──→ TRUE END / GOOD END
第八部 分岐点 ─────┤
                    ├── Stay 线（相守线）──→ NORMAL END
                    │
                    └── Bad 线（执念线）──→ BAD END
```

M/J 分流发生在第七部，影响该部的具体内容（默契 vs 较劲），但不改变最终路线。

### 4.3 四个结局
| 结局 | 条件 | 情感基调 |
|------|------|----------|
| **TRUE END** 💛 「世界第一，与你」 | Dream线 + nagiResonate + !antCompress + D≤3 + 多项旗标 | 他成为世界第一，也读懂了你 |
| **GOOD END** 🌸 「那么完美，那么爱你」 | Dream线 + !antCompress | 完美情侣，但缺少灵魂深处的共鸣 |
| **NORMAL END** 🧸 「普通情侣」 | Stay线 | 没有世界第一，但有慵懒甜蜜的每一天 |
| **BAD END** 🥀 「远处的世界第一」 | Cling路径 或 D≥6 | 你把他送到顶峰，但他不再回头 |

### 4.4 TRUE END 彩蛋
- 达成 TRUE END 后解锁「他的求婚」彩蛋剧情
- 可从结局页面和结局图鉴重复进入

## 5. 页面 & 画面系统

### 5.1 页面流转
```
主页 (home)
  ├── 开场白 (prologue) — 8句逐句展示，可跳过
  ├── 名字录入 (setup) — 首次游玩才出现
  └── 章节目录 (map) — 主导航
        ├── 展开章节 → 幕列表（可跳转已看过的幕）
        ├── 存档记录 (saves)
        ├── 结局图鉴 (gallery)
        └── 继续游戏 / 重新开始
```

### 5.2 游戏界面层级
```
底层: 背景图 (gamebg) — 全屏铺满，center top / cover
中层: 遮罩层 (gameveil) — 半透明渐变
顶层:
  ├── HUD（返回、存档、章节名、羁绊条）
  ├── 对话面板 (dialog) — 底部，说话者名字 + 对话文字 + 打字指示器
  ├── 旁白面板 (narration) — 全屏蒙版 + 居中多段文字
  ├── 选项面板 (choices) — 居中卡片列表
  ├── 节标题卡 (chapter) — 章名 + 节名，轻触继续
  ├── 节完成卡 (clear) — 本幕完成 / 回目录 / 下一幕
  └── 结局卡 (ending) — 结局 emoji + 标题 + 描述 + 路线信息
```

### 5.3 背景系统
- ~50 张场景背景图（`bg/` 目录）
- 每个场景节点可指定独立背景 (`NODEBG` 映射)
- 章节也有默认背景，节点未指定时回退到章背景，再回退到 `bg/apartment.jpg`
- 背景切换无过渡（直接替换 `backgroundImage`）

### 5.4 视觉风格
- **色系**: 粉玫瑰系（`#fdf6f6` 奶油底色，`#c25670` 玫瑰色强调）
- **字体**: 衬线字体用于标题/章节（Baskerville/宋体），无衬线用于正文
- **玻璃拟态**: 大量 `backdrop-filter: blur()` + 半透明背景
- **装饰**: 花瓣飘落动画（11片，循环下落 + 旋转）
- **整体尺寸**: 移动端优先，max-width 460px，max-height 940px

## 6. 存档系统

### 6.1 自动保存
- 游戏状态实时存入 `localStorage`（键名 `nagi_state`）
- 保存内容：当前节点ID、所有数值变量、所有旗标、路线标记

### 6.2 手动存档（多卡槽）
- HUD 中的💾按钮触发快速存档
- 最多保存 **30 条**存档
- 每条存档记录：节点位置、场景名、章幕信息、时间戳、所有游戏状态、玩家名字
- 存档列表页可查看/读取/删除
- 读取存档完整恢复所有游戏状态

### 6.3 进度追踪
- `seen` Set — 记录已查看的节点（控制章节目录中的可见性）
- `doneSet` — 记录已完成的幕（ci-si 格式）
- `unlocked` — 已解锁的最大章节索引
- 章节完成判定：该章所有幕都在 doneSet 中

### 6.4 结局收藏
- 达成结局后自动记录到 `nagi_endings`
- 结局图鉴展示 4/4 收集进度
- 未解锁结局显示为🔒

- 章标题

## 10. 节点数据结构（引擎核心）

### 10.1 场景节点 Schema
```javascript
{
  // 自动播放的对话序列
  auto: [
    { who: "sys" | "nagi" | "ant" | "yiyi", t: "台词文本" }
  ],
  
  // 玩家选项
  choices: [
    {
      label: "选项按钮文本",
      type: "心动" | "理性" | "作",    // 可选，标签分类
      ant: "玩家说的台词",             // 选择后播放
      
      // 数值变化
      b: 2,        // 羁绊 +2
      u: 1,        // 理解 +1
      i: 3,        // 懂他 +3（直接加 I）
      d: 1,        // 心墙 +1
      ego: 1,      // Nagi自主意志 +1
      control: 1,  // Ant管控 +1
      
      // Nagi 回应序列（字符串 = nagi台词，对象 = 指定角色）
      nagi: [
        "普通Nagi台词",
        { who: "sys", t: "旁白" },
        { who: "ant", t: "Ant插话" }
      ],
      
      // 流转控制
      next: "node_id",    // 跳转到指定节点
      sec: true,          // 本节结束
      end: true,          // 进入结局判定
      endTier: "bad",     // 直接指定结局等级
      endBad: true,       // 强制 Bad End
      egg: true,          // 彩蛋结束
      
      // 旗标设置
      route: "dream",     // 设置路线
      flag: "support",    // 设置终局标记
      set: {              // 批量设置变量
        line: "M",
        path: "dream",
        nagiResonate: true,
        antCompress: false,
        // ...
      }
    }
  ],
  
  // 路由节点（无 auto/choices，纯分流逻辑）
  router: true
}
```

### 10.2 章节配置 Schema
```javascript
{
  n: "第一部",
  title: "原作前置篇",
  bg: "bg/teamV.jpg",        // 章默认背景
  sections: [
    {
      n: "第一幕",
      title: "作战室·初遇",
      start: "p1",            // 入口节点 ID
      bg: "bg/pitch.jpg",     // 幕级背景覆盖（可选）
      routed: true,           // 是否有路线变体
      startStay: "stay_xxx",  // Stay 线替代入口
      titleStay: "...",       // Stay 线替代标题
      titleDream: "...",      // Dream 线替代标题
    }
  ]
}
```

### 10.3 节点链接 (FN — Flow Next)
- `FN` 对象定义默认的节点流转：`{ p1: "p2", p2: "c1a", ... }`
- `FN_PATH` 定义路线专属的跳转：`{ nodeId: { dream: "a", stay: "b" } }`
- `SEC_STARTS` Set 标记所有「幕起始节点」，用于自动切换幕/章计数

### 10.4 路由节点
- `resolveRouter(id)` 函数根据当前数值和旗标决定分流方向
- 例：`m_igate` 根据 control/D/ego/I 决定走深层亲密还是浅层客气
